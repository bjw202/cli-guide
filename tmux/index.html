01 · 개념

창문이 많은 방, 그리고 창문을 더 만들 수 있는 책상

tmux는 한 화면 안에서 여러 화면을 동시에 띄워두는 도구예요. 먼저 비유로 시작할게요.

컴퓨터 화면 하나를 '방'이라 상상해 보세요. 방에는 창문이 여러 개 달릴 수 있고, 창문 하나하나에는 서로 다른 풍경이 보여요. tmux는 이 방 안에서 창문을 마음껏 늘리거나, 나누거나, 다른 쪽으로 돌려놓을 수 있게 해 주는 '창문 관리 도구'예요.

비유를 조금 옮겨보면 이렇게 대응돼요. 방이 터미널 창(window), 창문 안의 조각이 pane, 그리고 창문을 달았다 뗐다 하는 행위가 바로 detach/attach예요. 이 방을 통째로 관리하는 큰 시스템이 터미널 멀티플렉서(terminal multiplexer)이고, tmux는 그중에서도 가장 사랑받는 한 종류랍니다.

옛날 이야기

옛날에는 screen이라는 도구가 먼저 있었어요. 1987년에 태어난 이 도구의 아이디어를 2007년에 Nicholas Marriott가 더 현대적으로 다듬어 태어난 게 바로 tmux(티뭉스, tee-mux)랍니다. 약 20년 가까이 쓰인 검증된 도구예요.

이렇게 생각해보세요

terminal / shell / multiplexer가 헷갈린다면 — terminal은 '창문 틀', shell은 '창문 너머로 말을 거는 비서', multiplexer는 '창문 틀을 통째로 관리하는 건축가'라고 생각하면 쉬워요.

알아두면 좋은

터미널은 글자만 보이는 검은/회색 화면이에요. 마우스를 잘 안 쓰고 키보드로 모든 걸 합니다. 처음엔 어색하지만, 익숙해지면 마우스보다 빠르고 편해져요.

02 · 왜 쓰나?

왜 다들 tmux를 좋아할까요?

다섯 가지 상황만 보면, "아, 이래서 쓰는구나"가 한 번에 와닿아요.

01

지속되는 작업

방에 불을 켜둔 채로 밖에 나갈 수 있어요. 인터넷이 끊겨도 작업은 방 안에 살아 있어요.

02

화면 분할

한 창에서 코드도 보고, 설명서도 보고, 채팅도 볼 수 있어요.

03

원격 작업 생존

노트북을 덮어도 서버에선 작업이 계속돼요.

04

페어 프로그래밍

같은 화면을 두 사람이 같이 볼 수 있어요.

05

멀티태스킹

여러 창을 번갈 보며 한 번에 여러 일을 해요.

이렇게 생각해보세요

옛날 컴퓨터는 화면이 하나뿐이었어요. 한 번에 하나만 할 수 있었죠. tmux는 그 하나의 화면을 여러 조각으로 쪼개 쓰는 아이디어에서 출발했어요.

알아두면 좋은

개발자들이 tmux를 사랑하는 진짜 이유는 '맥락(context)을 잃지 않는 것'이에요. 무엇을 하고 있었는지, 어디까지 했는지 화면이 그대로 남아 있으니까요.

03 · 설치

설치해 봐요 (3분 컷)

운영체제에 맞춰 한 줄만 복사하면 돼요. 그 전에 'prefix 키'라는 핵심 개념도 짚고 갈게요.

mac

macOS

Homebrew라는 '앱스토어 같은 도구'로 한 줄 설치.

brew install tmux
Linux

Ubuntu / Debian

리눅스의 앱스토어 명령어(apt)로 설치.

sudo apt install tmux
Windows

Windows (WSL)

윈도우 안에 작은 리눅스 방(WSL)을 하나 더 달아서 그 안에서 tmux를 써요. WSL 설치 후엔 리눅스와 동일.

sudo apt install tmux

핵심 개념: prefix 키 — tmux한테 말 거는 '종'

tmux한테 말을 걸려면 먼저 '종을 울려야' 해요. 그 종이 바로 prefix 키예요. 기본 종은 Ctrl+b 를 동시에 누르는 C-b 랍니다. 종을 울린 뒤에야 "창을 만들어줘(c)"처럼 명령을 내릴 수 있어요.

이렇게 생각해보세요

종(prefix)을 먼저 울리고 → 그 다음 명령을 하나 내리는 2단계 구조예요. "딸깍-딸깍" 두 번의 소리가 나면 그제야 tmux가 알아듣죠.

잠깐, 주의!

C-b+c 는 'Ctrl과 b를 누르고, 손을 뗀 뒤, c를 누른다'는 뜻이에요. Ctrl을 누른 채로 c까지 누르는 게 아니에요! 처음엔 다들 헷갈려요.

04 · 머릿속 그림

tmux의 4층 구조, 비유로 한 번에 정리

tmux 안에는 4개의 개념이 층층이 쌓여 있어요. 헷갈리지 않게, '건물'에 빗대어 볼게요.

tmux 용어일상 비유한 줄 설명
server건물의 관리사무소보이지 않는 곳에서 모든 걸 총괄. 전원이 꺼지지 않는 한 계속 돌아감.
session건물 안의 방 하나독립된 작업 공간. 방 여러 개를 만들 수 있고, 방마다 다른 일을 함.
window방 안의 책상한 방에 책상(탭)이 여러 장. 번호(0,1,2…)로 구분.
pane책상 위의 쪽지들한 책상을 여러 조각으로 쪼갠 각 조각. 각 조각마다 다른 프로그램이 돌아감.
건물 전체server

보이지 않는 관리사무소. 컴퓨터를 끄지 않는 한 항상 돌아가고 있어요.

tmux server
방session

건물 안의 독립된 방. 여러 개 만들 수 있고, 각자 다른 일을 해요.

session "work" session "logs" session "chat"
책상window

방 안의 책상(탭). 번호(0,1,2…)로 구분돼요.

0: editor 1: shell 2: logs
쪽지pane

책상 위를 여러 조각으로 쪼갠 각 조각. 각 조각이 화면의 한 영역이에요.

pane 0: vim pane 1: server pane 2: tests pane 3: git
이렇게 생각해보세요

session을 방, window를 책상이라 하면, detach는 '방에서 나와 현관문을 닫는 것', attach는 '다시 현관문을 열고 들어가는 것'이에요. 방 안의 물건(작업)은 그대로 남아 있어요.

알아두면 좋은

방(session)에 불을 끄지 않는 한, 그 안에서 돌아가는 프로그램은 계속 돌아가요. 이게 tmux의 가장 강력한 점이에요. active pane = 지금 손을 대고 있는 쪽지 / current window = 지금 앉아 있는 책상.

세션을 시작하려면 아래 "세션 시작" 버튼을 눌러주세요.
팁: 실제로는 C-b 가 prefix(종)이고, 그 다음 키가 명령이에요.
꼭 알아두면 좋아요

번호가 헷갈릴 땐 C-b+q! 각 쪽지 위에 큰 숫자가 잠깐 뜨면서, 그 숫자를 누르면 바로 그 쪽지로 이동해요.

잠깐, 주의!

C-b+d 로 나가도 방(session)은 절대 사라지지 않아요. '닫은 게 아니라 잠시 나온 것'이에요. 다시 tmux a 치면 그대로 있어요.

이렇게 생각해보세요

화면 분할은 책상을 자르는 것과 같아요. 같은 책상 위에 두 조각이 나란히 놓이는 거죠. 한 조각에서 글을 쓰면서 다른 조각의 자료를 볼 수 있어요.

07 · 복사 모드

마우스 없이 글자를 복사하고, 위로 스크롤해요

터미널에서 위로 스크롤하려고 마우스 휠을 굴려도 안 움직여서 당황한 적 있죠? tmux에서는 '복사 모드'라는 특별한 모드로 들어가야 해요.

  1. 1
    복사 모드 진입 — C-b+[ 돋보기를 꺼내 드는 단계. 이제 화면이 "정지"해요.
  2. 2
    이동 — 화살표 / Page Up / Page Down 원하는 위치로 자유롭게 커서를 옮겨요.
  3. 3
    선택 & 복사 — (emacs 모드) C-Space 시작 → 이동 → C-w 복사 vi 모드라면 Space 시작, Enter 복사.
  4. 4
    붙여넣기 — C-b+] 복사한 내용이 다시 터미널에 입력돼요.
데모 · 복사 모드 한눈에 보기

버튼을 누르면 복사 모드 진입 → 선택 → 복사 흐름이 재생돼요.

복사 모드는 '잠시 화면을 정지시키고 돋보기로 살펴보는 시간'이에요. 보통 모드에서는 새 글자가 계속 올라오지만, 복사 모드에선 화면이 멈춰서 자유롭게 훑어볼 수 있어요.

잠깐, 주의!

복사 모드를 끝내려면 q! 빠져나오는 법을 잊으면 당황하기 쉬워요.

꼭 알아두면 좋아요

마우스를 쓰고 싶다면 .tmux.conf에 set -g mouse on을 한 줄 추가하면 휠 스크롤과 드래그 복사가 가능해져요.

알아두면 좋은

복사한 글자는 tmux만의 '버퍼'에 보관돼요. 최대 50개까지 저장되고, C-b+= 로 과거 버퍼 목록을 볼 수 있어요.

08 · 설정

나만의 tmux로 꾸미기

tmux는 집처럼 꾸밀 수 있어요. 집의 설계도 역할을 하는 파일이 바로 홈 폴더의 ~/.tmux.conf예요.

가장 자주 바꾸는 4가지: (1) prefix 키, (2) 마우스 켜기, (3) 상태표시줄 색/내용, (4) 창 번호를 1부터. 아래 코드는 주석을 달아뒀어요. 한 줄씩 읽어보면 "아 이걸 바꾸는 거구나"가 보여요.

# 1. prefix를 C-a로 바꾸기 (손이 더 편함) unbind C-b set -g prefix C-a bind C-a send-prefix # 2. 마우스 켜기 (휠 스크롤 + 드래그 복사) set -g mouse on # 3. 창 번호를 1부터 시작 (0은 너무 멀어요) set -g base-index 1 set -g pane-base-index 1 # 4. 상태표시줄 꾸미기 set -g status-style "bg=#2D2A26, fg=#FFB089" set -g status-left "#[fg=#FF8A5B,bold] #S " set -g status-right "#(whoami) · %H:%M"

상태표시줄(status line), 비교해 보기

화면 맨 아래 줄이 바로 상태표시줄이에요. 왼쪽엔 방 이름, 가운데엔 책상 목록, 오른쪽엔 시간이 떠요. 이것도 전부 내 마음대로 바꿀 수 있어요. 탭으로 둘 중 하나를 골라보세요.

기본 — tmux를 처음 켰을 때
$ vim app.js
작업 중...
[0] 0:bash*   "my-machine" 14:32

익숙한 tmux 기본 초록줄. 정보는 있지만, "내 것" 같은 느낌은 아직 없어요.

커스텀 — .tmux.conf를 고친 후
$ vim app.js
작업 중...
🟧 work · 1:editor* 2:shell 3:logs jw · 14:32

방 이름(work)이 크게 보이고, 색이 집 전체 톤과 어울려요. "내 방" 느낌이 살아요.

꼭 알아두면 좋아요

설정 파일을 고치고 나서 바로 적용하려면 tmux 안에서 C-b+: 로 명령 프롬프트를 열고 source ~/.tmux.conf를 입력하면 돼요. tmux를 다시 켤 필요 없어요.

알아두면 좋은

tmux 설정은 4가지 종류(server/session/window/pane)가 있어요. 보통은 전체에 적용되는 global 설정(-g)만 만져도 충분해요.

저장
tmux-resurrect

방 안의 책상 배치까지 기억했다가, 컴퓨터를 껐다 켜도 그대로 복원해 주는 '저장' 기능이에요.

자동
tmux-continuum

resurrect가 저장한 걸 주기적으로 자동 저장해 주는 비서. 컴퓨터가 꺼져도 마지막 상태가 살아 있어요.

알아두면 좋은

resurrect와 continuum을 같이 쓰면, "내 노트북이 꺼져도 서버 작업은 살아있다"는 약속이 진짜로 현실이 돼요. tmux의 궁극의 무기예요.

잠깐, 주의!

플러그인이 너무 많으면 tmux가 느려질 수 있어요. 정말 필요한 것부터 하나씩!

10 · 실생활 응용

실제로 어떻게 쓰이나요? 다섯 가지 이야기

지금까지 배운 개념들이 "아하"로 이어지는 클라이맥스 구간이에요.

01

SSH 원격 세션 지속

회사 서버에 접속해 작업하다 와이파이가 끊겨도, 다시 접속해서 tmux a 한 줄이면 그 자리로 복귀.

02

페어 프로그래밍

두 사람이 같은 tmux session에 붙어서 한 화면을 같이 보며 코딩.

03

로그 모니터링

한 화면에 서버 로그/에러/현재 작업을 동시에 띄워두고 한눈에 감시.

04

멀티 모니터 대체

모니터가 하나여도 tmux 분할로 여러 창을 동시에.

05

노트북 꺼져도 OK

클라이맥스. 노트북을 덮고 퇴근해도, 서버의 tmux session은 계속 돌아가고 있어요.

"와이파이가 끊겼을 때" — 일반 터미널 vs tmux

같은 상황인데 결과가 완전히 달라요. 한 번 보면 잊을 수가 없어요.

✗ tmux 없이 일반 터미널
$ ssh server
$ npm run build
> building...
[연결 끊김]
↳ 프로세스 사라짐. 처음부터 다시.
✓ tmux로 감싼 경우
$ ssh server
$ tmux
$ npm run build
[연결 끊김 → 다시 접속]
$ tmux a
↳ 그 자리 그대로. 빌드 계속됨.
이렇게 생각해보세요

tmux 없이 SSH를 쓰면, 인터넷이 끊기는 순간 서버에서 하던 일도 같이 사라져요. 마치 전화를 끊으면 통화 내용이 증발하는 것처럼요. tmux는 '통화 내용이 녹음되어 남는 전화'를 만들어 주는 도구예요.

알아두면 좋은 — 보안

공유 session은 강력하지만 주의가 필요해요. 같은 session에 붙은 사람은 내가 치는 모든 걸 볼 수 있어요. 믿을 수 있는 사람과만!

11 · 문제 해결

당황하지 마세요, 다 해결책이 있어요

초보자가 가장 많이 겪는 6가지 상황. 그리고 가장 흔한 공포를 직접 풀어보는 인터랙션까지.

해결책

기본 상태예요. tmux는 휠 스크롤을 가로챈 거예요. C-b+[ 로 복사 모드에 진입한 뒤 스크롤하세요. 또는 .tmux.conf에 set -g mouse on을 추가하면 휠 스크롤이 바로 됩니다.

해결책

mouse 옵션이 꺼져 있어서 그래요. .tmux.conf에 set -g mouse on 한 줄이면 휠 스크롤, 클릭으로 pane 이동, 드래그 복사까지 다 됩니다.

해결책

터미널이 256색을 쓴다고 못 알아들은 거예요. .tmux.conf에 set -s default-terminal 'tmux-256color' 추가, 그리고 필요하면 set -ga terminal-overrides ',*:Tc'로 트루컬러까지 켜주세요.

해결책 — 그리고 직접 체험해 보세요

아니요! 방(session)은 그대로예요. tmux a를 치면 그 자리로 돌아갑니다. 가장 흔한 초보 공포 — "detach 해버렸다" — 를 아래서 직접 겪어보고 풀어보세요.

detach 공포 — 안심 인터랙션
$ vim memo.txt
오늘 할 일: ...
1. tmux 세션 만들기 ✓
2. detach 해보기
😌
별거 아니었죠? 작업은 그 자리에 그대로였어요.
이게 바로 tmux의 약속이에요. detach는 '나가기'가 아니라 '잠시 밖에 나가기'일 뿐.
해결책

먼저 tmux ls로 현재 열린 방 목록을 확인하세요. 이름을 알면 tmux a -t 이름으로 그 방으로 정확히 재진입할 수 있어요.

해결책

tmux kill-server를 치면 건물(서버) 전체가 종료돼요. 단, 모든 세션과 작업이 즉시 사라지니 정말 다 필요 없을 때만!

잠깐, 주의!

kill-server는 건물(서버) 자체를 허무는 거예요. 모든 방과 작업이 즉시 사라져요. 정말 다 필요 없을 때만!

꼭 알아두면 좋아요

tmux 안에서 뭔가 이상하면 C-b+?(단축키 목록), C-b+:(명령 프롬프트)로 항상 도움을 받을 수 있어요.

12 · 더 깊이 (보너스)

더 깊이 알고 싶은 분을 위해

곁가지 주제를 한 곳에 모았어요. 안 읽어도 tmux는 잘 쓸 수 있지만, 읽으면 더 재미있어요.

tmux는 2007년 OpenBSD 운영체제 개발 과정에서 태어났어요. "화면 관리 도구가 더 필요하다"는 요구에서 시작됐죠. 그 전에는 screen(1987년 탄생)이 표준이었어요.

tmux는 screen의 핵심 아이디어(세션 지속, 분할)를 계승하면서도 설정 파일, 스크립트 기반 자동화, 플러그인 생태계를 더 현대적으로 다듬었어요. 그 결과 지금은 tmux가 사실상 표준.

개발은 한 번에 수십 개의 "맥락"을 동시에 다뤄요 — 서버 로그, 에디터, 터미널, 문서. 이걸 매번 켜고 끄면, "아까 뭘 보고 있었더라?"로 에너지가 새어나가요.

tmux는 그 맥락 전체를 '방'에 보존해요. 다음 날 아침에 tmux a 한 번이면 어제의 머릿속이 그대로 복원돼요. 이게 "맥락 보존(context preservation)" 철학이고, tmux 사랑의 진짜 이유예요.

요약: 서버에서 쓸 거면 tmux. 차세대 감성이 좋다면 zellij. 로컬 mac에서만 쓴다면 iTerm2 탭도 훌륭해요.

두 사람이 같은 session에 붙으면(페어 프로그래밍), 서로가 치는 모든 키와 화면을 공유해요. 강력하지만 그만큼 위험해요 — 비밀번호, 토큰, 개인정보도 다 보여요. 믿을 수 있는 팀원과만 공유하고, 민감 작업은 별도 session으로 빼는 걸 추천해요.

도구는 강력하지만, 도구 그 자체로 생산적이지는 않아요. "화면을 분할하기 전에, 내가 지금 무엇에 집중하고 있는지"를 먼저 묻는 습관이 tmux를 더 빛나게 해요. 도구는 생각의 확장일 뿐, 대체재가 아니에요.

옛날 이야기

알아두면 좋은

요즘은 zellij라는 '차세대 tmux'를 만들려는 시도도 있어요. 하지만 tmux는 20년 가까운 검증과 엄청난 커뮤니티 자료를 가진, 여전히 사실상 표준이에요.
