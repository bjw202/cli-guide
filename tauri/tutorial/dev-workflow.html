튜토리얼 · T04 · 흐름

개발 흐름 한눈에 — 요리법

재료 준비(scaffold) → 시식하며 개발(dev) → 완성품 포장(build). Tauri 개발은 요리법처럼 정해진 단계를 따라요.

비유

요리를 상상해 보세요. 재료 준비(scaffold) → 시식하며 개발(dev, 저장하면 바로 갈아줌) → 완성품 포장(build). Tauri CLI가 이 요리법을 한 줄씩 실행해 줘요.

🔁 흐름 — 재생하거나 각 단계를 눌러 보세요

명령 3개로 끝나는 개발

# 1. 앱 뼈대 만들기 (scaffold)
npm create tauri-app@latest

# 2. 개발 — 핫 리로드 (프론트 + Rust 동시)
cd my-app
npm run tauri dev

# 3. 빌드 — 운영체제별 인스톨러
npm run tauri build

사실 이게 핵심이에요. 3개 명령으로 앱 하나가 만들어져요. 나머지는 프론트엔드 짜듯 화면을 만들고, Rust에 명령을 추가하는 일이에요.

팁

npm run tauri dev를 켜둔 채 코드를 고치면, 저장 즉시 화면이 갱신돼요. 프론트는 Vite가, Rust는 cargo가 각자 핫 리로드해요 — 두 세계가 동시에 "뜨거운" 상태를 유지해요.

조심하세요

처음 cargo tauri dev는 Rust 의존성을 컴파일하느라 몇 분 걸려요. 이후엔 캐시가 있어 빨라요. "멈춘 것 같다" 싶어도 기다리세요 — Rust 컴파일은 첫 번째가 가장 느려요.
