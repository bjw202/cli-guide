왜 Tauri인가 — 가벼움이라는 답

2013년 Electron이 웹으로 데스크톱 앱을 만들었지만, "무겁다"는 대가가 있었어요. 2019년 Tauri는 그 문제에 "운영체제의 창을 빌려 쓰자"고 답했어요.

비유

Electron은 앱마다 브라우저(Chromium)를 한 벌씩 들고 다닌 거예요. Tauri는 운영체제가 이미 깔아둔 창을 빌려 쓰고, 무거운 짐은 Rust에게 맡겨요. 같은 "웹으로 만든 앱"인데 배낭을 메고 다니느냐, 가볍게 손만 들고 가느냐의 차이예요.

역사

2013 GitHub, Electron 발표(웹 기술 → 데스크톱). VS Code·Slack·Discord 탄생. · 2019 Daniel Thompson-Yvetot와 크루 커뮤니티, Tauri 시작. · 2020 Tauri v1. · 2024 Tauri v2(모바일 iOS/Android 지원, ACL 보안 강화).

문제의식 — Electron은 "무겁다"

Electron의 혁명에는 대가가 따랐어요. 모든 앱에 Chromium 브라우저와 Node.js 런타임을 통째로 포함했거든요. 그 결과:

  • 설치 파일 크기: 보통 100~200MB (간단한 메모 앱조차).
  • 메모리 사용: 100~300MB 시작, 창이 늘면 더.
  • 배터리·자원: 앱마다 Chromium을 한 벌씩 켜는 셈.

"웹 기술로 만들 수 있다"는 장점이 "무겁고 자원을 많이 먹는다"는 단점과 늘 붙어 다녔어요.

Tauri의 답 — 창을 빌려 쓰고, 일은 Rust에게

Tauri의 발상은 단순했어요. 두 가지:

  1. 운영체제가 이미 가진 웹뷰를 쓴다. Windows는 WebView2, macOS는 WKWebView, Linux는 WebKitGTK — 운영체제에 기본 탑재된 웹 렌더러예요. Chromium을 번들하지 않아도 됩니다. 앱 크기가 3~10MB로 쪼그라들어요.
  2. 백엔드를 Rust로 짠다. Node.js 대신 Rust — 시스템 언어라 빠르고 메모리 안전. 결과물은 진짜 네이티브 바이너리(JS 런타임 없음). 메모리 30~80MB 수준.
// Electron: Chromium + Node.js를 앱마다 통째로 포함 (~150MB)
// Tauri  : OS 웹뷰 + Rust 백엔드 (~5MB) — 가벼운 배낭 대신 손만 들고

왜 쓰나 — 4가지

  • 가볍다: 설치 파일 수 MB, 메모리 1/3~1/5. Electron 앱 여러 개 켜둔 느낌과 다르다.
  • 안전하다: Rust가 메모리 안전. Tauri 2는 권한 기반(ACL)으로 "최소권한" 원칙.
  • 크로스플랫폼 + 모바일: 하나의 코드로 Windows·macOS·Linux, 그리고 (v2) iOS·Android까지.
  • 웹 기술 그대로: React·Vue·Svelte·vanilla — 이미 아는 것으로 화면을 만든다.
이렇게도 써요

Tauri로 만든 앱: 1Password, Sourcegraph Cody, Pomofocus, 그리고 수많은 오픈소스 도구. 가벼움이 중요한 도구·유틸리티 앱에서 특히 빛을 발해요.

팁

"가벼움"이 Tauri의 가장 강력한 카드지만 유일한 장점은 아니에요. 보안(Rust + ACL)과 모바일까지 하나의 코드베이스라는 점도 Electron과의 분명한 차이예요. 더 자세한 비교는 다음 페이지(T02)에서.
