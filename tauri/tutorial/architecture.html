3계층 아키텍처 — 식당으로 이해하기

Tauri 앱은 손님(프론트엔드)이 주문서(invoke)를 주방(Rust)에 넘기고, 요리(command)를 받아 식당 홀(OS 웹뷰)에서 즐기는 구조예요.

비유

식당을 상상해 보세요. 손님이 메뉴판(프론트엔드)을 보고 주문서(invoke)를 카운터에 놓아요. 주방(Rust 백엔드)이 주문을 받아 요리(`#[tauri::command]`)하고, 완성된 음식을 손님에게 내어줘요. 식당 홀은 운영체제의 웹뷰 — 음식이 보이는 공간이에요.

🏗️ 3계층 — 클릭하면 설명이 나와요

각 층을 눌러보세요. 손님→주문서→주방이 어떻게 이어지는지 보여드려요.

↕ ↕
각 층을 클릭하면 자세한 설명이 나와요.

1층 — 프론트엔드 (웹)

화면은 이미 아는 웹 기술로 만들어요. HTML/CSS/JS, 그리고 React·Vue·Svelte 같은 프레임워크. "손님"이 보는 메뉴판이에요. 웹이라서 배운 것이 그대로 쓰이고, 디자인도 웹처럼 자유로워요.

2층 — IPC (주문서)

손님이 주방에 직접 들어가 요리하지 않듯, 프론트엔드가 Rust 코드를 직접 실행하진 않아요. 대신 주문서를 넘겨요. Tauri에선 이걸 invoke라고 불러요.

// 프론트엔드(JS) — 주문서를 넘김
const result = await invoke('greet', { name: '지웅' });

3층 — Rust 백엔드 (주방)

주방은 Rust로 짜요. 주문서(invoke)가 들어오면 해당 명령(command)이 실행돼요. 파일 읽기·네트워크·계산 같은 "진짜 일"은 여기서 일어나요. Rust라 빠르고 안전해요.

// 백엔드(Rust) — 주방이 요리
#[tauri::command]
fn greet(name: String) -> String {
    format!("안녕, {}!", name)
}
알아두면 좋은

4번째 요소가 하나 더 있어요 — 운영체제의 웹뷰(식당 홀). 프론트엔드가 그려지는 "창"이에요. Tauri는 이 창을 운영체제 것을 빌려 쓰기 때문에 가벼운 거예요(T01 참고).
