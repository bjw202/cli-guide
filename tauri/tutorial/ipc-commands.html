튜토리얼 · T07 · IPC

프론트와 Rust가 대화하는 법 — IPC

손님이 직접 주방에 들어가 요리하지 않듯, 화면은 Rust 코드를 직접 실행하지 않아요. 주문서(invoke)를 넘기고, 주방 명령(command)이 응답해요.

비유

식당(T03)을 떠올려 보세요. 주문서(invoke) → 주방 명령(command). 프론트엔드가 invoke('greet', { name })로 주문을 넣으면, Rust의 #[tauri::command] fn greet가 요리해서 결과를 내어줘요.

📨 직접 주문 넣어보기

이름을 넣고 실행하면, 프론트 invoke → Rust command → 결과 흐름을 보여드려요.

손님 이름

// 버튼을 누르면 주문이 시작돼요

주문서와 명령의 쌍

프론트와 Rust는 이름('greet')으로 짝을 맞춰요. 프론트가 invoke('greet')로 부르면, Rust의 #[tauri::command] fn greet가 반응해요.

// 백엔드(Rust) — 명령 정의
#[tauri::command]
fn greet(name: String) -> String {
    format!("안녕, {}!", name)
}

// 프론트엔드(JS) — 명령 호출
import { invoke } from '@tauri-apps/api/core';
const msg = await invoke('greet', { name: '지웅' });
console.log(msg); // "안녕, 지웅!"

명명단에 등록하기

명령을 만들었다고 끝이 아니에요 — Tauri 앱 시작 지점에 "이 명령을 쓸 수 있게 해줘"고 등록해야 해요.

tauri::Builder::default()
    .invoke_handler(tauri::generate_handler![greet])
    .run(tauri::generate_context!())
    .expect("error while running tauri application");
팁 — 최소권한 (Tauri 2)

Tauri 2는 권한 기반(ACL)이에요. 모든 명령·API는 명시적으로 허용해야 쓸 수 있어요. "기본으로 다 허용"이 아니라 "기본으로 다 막힘" — 그래서 안전해요. 권한은 capabilities/ 설정에서 관리해요.

IPC 핵심

invoke('이름', args)프론트 → Rust 주문. await로 결과 받음.
#[tauri::command]Rust 함수를 프론트에 노출.
generate_handler![...]앱 시작 시 명령 명단 등록.
ACL (Tauri 2)최소권한 — 명시적 허용 필수. 기본은 차단.
