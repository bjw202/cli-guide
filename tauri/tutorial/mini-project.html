튜토리얼 · T09 · 실전 (터미널)

실전 미니 — "안녕, {이름}!" 앱

배운 것 전부를 한 회로로 엮어요: scaffold → Rust 명령 → 프론트 invoke → 빌드. 가장 작지만 온전한 Tauri 앱이에요.

비유

이제 직접 주문을 넣어보세요. 손님이름을 적어 주문서(invoke)를 주방(Rust)에 보내면, 주방이 "안녕, {이름}!" 하고 요리해 돌려줘요. Tauri의 전체 회로가 이 한 흐름에 다 들어 있어요.

🚀 직접 주문 넣어보기

실제 Tauri 없이도 회로를 체험할 수 있어요. 이름을 바꿔 보세요.

손님 이름

// 버튼을 누르면 주문이 시작돼요

5단계로 만드는 미니 앱

# 1. scaffold
npm create tauri-app@latest  # → vanilla 선택

# 2. Rust 명령 추가 (src-tauri/src/main.rs 또는 lib.rs)
#[tauri::command]
fn greet(name: &str) -> String {
    format!("안녕, {}!", name)
}

# 3. 명령 등록
.invoke_handler(tauri::generate_handler![greet])

# 4. 프론트에서 호출 (index.html 의 JS)
import { invoke } from '@tauri-apps/api/core';
const msg = await invoke('greet', { name: '지웅' });

# 5. 빌드
npm run tauri build
팁

이 5단계가 Tauri 앱의 전부라고 해도 과언이 아니에요. 실제 앱은 "명령 더 만들기 + 화면 더 꾸미기 + 권한 더 허용하기"의 반복이에요. 골격은 이게 전부.

알아두면 좋은

이름 대신 파일 읽기·시스템 호출·데이터베이스 같은 "진짜 일"을 Rust 명령에 넣으면, 그게 곧 실사용 앱이 돼요. 프론트는 그저 결과를 보여주는 역할이에요.

T09 한 장 요약

Tauri 앱 만들기 5단계

1. scaffoldnpm create tauri-app — 뼈대 생성.
2. 명령 작성#[tauri::command] fn ... — Rust에 기능 추가.
3. 명령 등록generate_handler![...] — 명단에 추가.
4. 프론트 호출invoke('이름') — 화면에서 부르기.
5. 빌드cargo tauri build — 인스톨러 생성.
