튜토리얼 · T08 · 빌드·배포

빌드와 배포 — 완성품 포장하기

개발이 끝나면 완성품을 포장해야 해요. 운영체제마다 다른 상자(.msi/.dmg/.deb)에 담아, 사용자가 설치할 수 있게 만드는 과정이에요.

비유

요리가 완성되면 포장해서 내보내야 해요. 윈도우 고객에게는 윈도우 상자, 맥 고객에게는 맥 상자 — 운영체제마다 다른 포장이 필요해요. Tauri가 이 포장을 한 번에 해줘요.

tauri.conf.json — 앱 명세서

빌드 전에 앱의 "명세서"를 적어요 — 창 크기, 앱 이름, 아이콘, 번들 식별자, 권한.

{
  "productName": "내 앱",
  "version": "0.1.0",
  "identifier": "com.example.my-app",
  "build": { "frontendDist": "../dist" },
  "app": {
    "windows": [{ "title": "내 앱", "width": 800, "height": 600 }]
  }
}

cargo tauri build — 포장 시작

명령 한 줄이면 운영체제별 인스톨러가 만들어져요. Rust는 릴리즈 최적화까지 알아서 해요.

npm run tauri build
# → src-tauri/target/release/bundle/ 에 산출물

산출물은 운영체제에 따라:

Windows.msi / .exe (NSIS) 설치 파일.
macOS.dmg / .app. (코드 서명/공증 권장.)
Linux.deb / .AppImage / .rpm.
이렇게도 써요 — 모바일

Tauri 2는 같은 코드베이스로 모바일까지 빌드해요. cargo tauri android build / cargo tauri ios build. Electron엔 없는 Tauri만의 강점 — 데스크톱·모바일을 하나의 코드로.

팁

앱 크기는 보통 3~10MB 수준이에요(T01 참고). Electron 대비 20~50배 가벼운 이유가 빌드 산출물에서도 드러나요. Chromium을 번들하지 않으니까요.
