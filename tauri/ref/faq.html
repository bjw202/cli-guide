자주 묻는 질문 (FAQ)

Tauri를 처음 접할 때 품게 되는 의문들. 솔직하게, 비유로 답해요.

Rust, 어렵지 않나요?

깊이 들어가면 어려워요(소유권·수명). 하지만 Tauri 앱을 만드는 데 필요한 Rust는 최소예요 — 본 가이드 T06의 7가지만 알면 예제가 읽혀요. 프론트엔드는 이미 아는 웹 기술 그대로고, Rust는 "명령 몇 개 짜기"로 시작해요. 점차 늘려가면 됩니다.

Tauri v1과 v2의 차이는?

v2(2024~)의 큰 변화: (1) 모바일(iOS/Android) 지원, (2) ACL 기반 최소권한 보안, (3) 새로운 플러그인 시스템. 본 가이드는 v2 기준이에요. 새 프로젝트는 무조건 v2로 시작하세요.

기존 Electron 앱을 Tauri로 옮길 수 있나요?

프론트엔드(웹)는 거의 그대로 쓸 수 있어요. 하지만 백엔드를 Node.js에서 Rust로 다시 짜야 해요. IPC(invoke/command) 패턴으로 옮기는 작업이 주된 몫이에요. 점진적 마이그레이션보단 "프론트 재사용 + 백엔드 재작성" 접근이 현실적이에요.

모바일 진짜 되나요?

Tauri 2에서 가능해요. cargo tauri android build / ios build. 다만 앱스토어 심사·코드 서명·네이티브 권한 등은 데스크톱과 다른 별도 과정이에요. 본 가이드는 "가능하다"까지 다루고, 심사 절차는 범위 밖이에요.

왜 Electron처럼 무겁지 않은 건가요?

운영체제가 이미 깔아둔 웹뷰를 빌려 쓰기 때문이에요. Electron은 앱마다 Chromium을 번들하지만, Tauri는 WebView2/WKWebView/WebKitGTK를 써요. 그래서 설치 파일이 수 MB. 백엔드도 Rust라 가벼운 네이티브 바이너리예요. (T01/T02 참고)

처음 cargo tauri dev가 왜 이렇게 느리죠?

첫 실행은 Rust 의존성을 컴파일하느라 몇 분 걸려요. 캐시가 만들어지면 이후엔 빨라요. "멈춘 것 같다" 싶어도 기다리세요. Rust 컴파일은 첫 번째가 가장 느려요. (T04 참고)

어떤 프론트엔드 프레임워크가 좋나요?

이미 아는 것으로 시작하세요. React·Vue·Svelte 전부 공식 템플릿이 있어요. npm create tauri-app이 선택지를 줘요. 처음이라면 vanilla(순수 HTML/JS)로 골격을 잡고 나중에 프레임워크를 얹어도 돼요.
