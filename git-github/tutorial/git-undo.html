튜토리얼 · T11 · 되돌리기

실수했다고요? 괜찮아요 — Git에서는 거의 다 되돌릴 수 있어요

이 페이지가 어쩌면 제일 중요할지도 몰라요. 비개발자가 Git을 두려워하는 가장 큰 이유가 '실수하면 다 날아가지 않을까'인데, 방법만 알면 Git은 꽤 안전한 편이에요.

이렇게도 써요

법률가도 '계약서 예전 버전으로 돌아가야 할 때' 버전 관리 덕에 안전하게 되돌려요. 그리고 실수 복구 패턴을 모아둔 'Oh Shit, Git!?!' 사이트는 전 세계 개발자의 구명조끼예요.

비유

Git의 사진첩은 한 번 찍은 사진을 안전한 쓰레기통(reflog)에도 복사해 둬요. 도달 가능한 사진은 기본 90일, 그렇지 않은 사진도 30일간 보관돼요. 쓰레기통을 비우지 않는 한 거의 다 꺼낼 수 있어요.

reset — 사진첩에서 사진 몇 장 떼어내기

git reset은 현재 위치에서 '몇 장 뒤로 갈지' 정하는 명령이에요. 세 가지 강도가 있어요.

--soft: HEAD만 뒤로. 사진은 떼어지지만 내 작업 방과 대기 방은 그대로예요. '방금 찍은 사진을 다시 고쳐 찍고 싶을 때'.
--mixed (기본값): HEAD를 뒤로 옮기고 대기 방(staging)까지 비워요. 작업 방 파일은 그대로. '대기 방에 잘못 올린 걸 내릴 때'.
--hard: HEAD를 뒤로 옮기고 대기 방과 작업 방까지 완전히 덮어써요. 커밋하지 않은 변경은 정말로 사라져요.

조심하세요

git reset --hard는 작업 방의 파일까지 덮어써요. 아직 사진으로 안 찍은(커밋 안 한) 변경은 정말 날아가요. 커밋하지 않은 변경이 있을 땐 절대 쓰지 마세요.

revert — '취소합니다'라는 새 사진 찍기

git revert <커밋>는 사진을 떼어내는 게 아니라, '이 사진 취소합니다'라는 취소 사진을 새로 찍는 거예요. 역사를 안 지우기 때문에 이미 다른 사람과 공개한 저장소에서 안전해요.

조심하세요

이미 push한 건 revert로, 아직 안 올린 건 reset으로. reset은 역사를 지워버리니까 공개된 저장소에 쓰면 동료의 사진첩이 꼬여요. 로컬에서만 reset, 공개 뒤엔 revert. 이 원칙만 기억하세요.

restore & clean — 파일 단위 되돌리기

git restore <파일>은 파일을 마지막 사진 상태로 되돌려요. git restore --staged <파일>는 대기 방에서 빼내 원래 작업 방으로 돌려요. (예전에는 git checkout <파일>로 썼어요.)

git clean -fd는 Git이 추적하지 않는 새 파일·빈 폴더를 삭제해요. 주의! 한 번 지운 건 reflog에도 안 남아요.

reflog — 잃어버린 사진 찾기

실수로 reset --hard를 해도 끝이 아니에요. git reflog는 '내가 최근에 머물렀던 모든 위치'를 기록해 둬요. 거기서 원래 사진의 해시를 찾아 git reset --hard <해시>로 다시 돌아가면 돼요. 쓰레기통에서 꺼내는 거랍니다.

알아두면 좋은

'Oh Shit, Git!?!' 패턴 — 자주 묻는 복구 시나리오:
• '잘못 브랜치에서 커밋했어요' → cherry-pick 또는 새 브랜치 만들고 reset.
• 'reset --hard 했는데 필요했어요' → reflog에서 복구.
• 'force push 해서 동료 작업 날아갔어요' → 정말 조심. reflog와 동료 협조로 일부 복구 가능.

🛟 "아 망할!" — 실수 복구 시뮬레이터

상황을 고르면 복구 명령과 "왜 작동하나"가 나와요. (ohshitgit.com 오마주)

T11 한 장 요약

되돌리기 핵심 명령

git reset --soft <커밋>HEAD만 뒤로. 작업·대기 방 유지.
git reset --mixed <커밋>기본값. HEAD 뒤로 + 대기 방 비움.
git reset --hard <커밋>HEAD + 대기 + 작업 방까지 완전히. 주의!
git revert <커밋>취소 사진을 새로 찍기. 공개 저장소에 안전.
git restore <파일>파일을 마지막 사진 상태로 되돌리기.
git restore --staged <파일>대기 방에서 빼내기.
git clean -fd추적 안 되는 새 파일·빈 폴더 삭제. 주의!
git reflog최근 머문 위치 기록. 잃어버린 커밋 찾기.
