튜토리얼 · T03 · Git 기초

세 단어의 마법 — status, add, commit

Git의 하루는 사실 세 단어로 끝나요. 지금 뭐가 바뀌었는지 보고(status), 바뀐 걸 하나로 묶어서(add), 사진첩에 한 장으로 붙이기(commit). 이 세 단계만 알면 Git의 절반은 한 거랍니다.

이렇게도 써요

소설가도 챕터가 끝날 때마다 원고를 '한 장'으로 묶어 저장하죠. 요리사도 레시피를 단계별로 정리해서 '오늘의 메뉴'로 묶어요. Git의 add·commit은 글쓰기와 요리 모두에 쓰이는 '한 단락 마무리하기' 흐름이에요.

Git 하루의 세 단어

복잡해 보이는 Git도, 매일 반복하는 핵심은 딱 세 단계예요. 먼저 큰 그림으로 보면:

  1. status — "지금 뭐가 바뀌었고, 사진첩에 안 붙인 게 뭐야?"
  2. add — "이 바뀐 파일, 다음 사진에 넣어줘."
  3. commit — "메시지를 적어서 사진첩에 한 장으로 붙여."

이 세 단어를 한 바퀴 도는 게 Git의 '기본 사이클'이에요. 처음엔 이 흐름을 손에 익히는 게 전부랍니다.

비유

add는 '한 묶음으로 포장하기', commit은 '포장한 묶음에 도장 찍어 사진첩에 붙이기'예요. 포장하기 전에는 얼마든지 뺄 수 있고, 도장을 찍은 뒤에는 사진첩에 영구히 남아요.

한 번에 보는 기본 사이클

아래 터미널을 천천히 스크롤해 보세요. 빈 사진첩을 만들고(git init), 파일이 생겼는지 확인하고(git status), 묶어서(git add) 사진첩에 붙이는(git commit) 흐름이 한눈에 펼쳐져 있어요.

터미널 — git 기본 흐름
스크롤하면 재생됩니다…

단계별로 풀어볼게요. git status는 지금 작업 방(working directory)의 상태를 보여줘요. 새로 만든 파일은 '추적 안 됨(Untracked)'로 빨간색으로 떠요. git add readme.md로 그 파일을 '다음 사진에 넣겠다'고 표시하면, 초록색으로 바뀌죠. 그리고 git commit -m "메시지"로 도장을 찍으면, 사진첩에 한 장으로 영구히 남아요.

팁

처음엔 git status를 아주 자주 치세요. Git이 친절하게 "지금 뭘 해야 할지"를 알려줘요. "이 파일은 아직 안 넣었어", "이건 넣을 준비 됐어" 같은 안내가 뜨니까, 막막할 땐 일단 git status부터.

붙인 사진 다시 보기 — log와 show

사진을 붙이고 나면, "내가 지금까지 뭘 붙였더라?"가 궁금해져요. 이때 두 명령이 나와요.

git log는 사진첩의 목차예요. 가장 최근 사진이 위에, 처음 사진이 아래에 나열돼요. git log --oneline을 붙이면 한 줄 요약으로 쫙 보여줘요. 터미널 데모의 마지막 줄이 바로 이 모습이에요.

git show는 사진 한 장을 확대해 보는 도구예요. git show만 치면 가장 최근 사진을, git show a1b2c3d처럼 특정 사진 번호(SHA)를 주면 그 사진을 자세히 보여줘요. 무엇이 바뀌었는지(diff)까지 같이요.

방금 찍은 사진의 메시지 고치기 — amend

커밋 메시지에 오타가 났거나, 내용이 마음에 안 들 때. 아직 사진첩에 붙인 직후라면 고칠 수 있어요.

git commit --amend -m "새로운 메시지"

이 명령은 '방금 찍은 사진 하나'를 새로 찍어 덮어쓰는 거예요. 사진첩의 다른 사진엔 영향이 없어요. 다만, 이미 인터넷(GitHub)에 올린 사진은 amend하면 안 돼요 — 다른 사람과 사진첩이 엇갈릴 수 있거든요. 자세한 건 T09(remote)와 T11(undo)에서 다뤄요.

잠깐, 주의!

커밋 메시지는 '무엇을 바꿨는지'보다 '왜 바꿨는지'를 적는 게 나중에 큰 도움이 돼요. "readme 수정"보다 "프로젝트 목적을 처음 방문자에게 명확히"가, 두 달 뒤의 나와 동료에게 훨씬 친절한 메시지랍니다.

T03 한 장 요약

수정 → 묶기 → 기록의 기본 사이클

git status지금 뭐가 바뀌었는지, 사진첩에 안 붙인 게 뭔지 보기.
git add <파일>바뀐 파일을 다음 사진에 넣겠다고 포장하기.
git commit -m "메시지"포장한 묶음에 도장 찍어 사진첩에 한 장으로 붙이기.
git log지금까지 붙인 사진 목록 보기 (--oneline 추천).
git show [커밋]특정 사진(기본=최근) 자세히 보기. diff까지.
git commit --amend방금 찍은 사진의 메시지/내용 고치기 (안 올린 사진만).
