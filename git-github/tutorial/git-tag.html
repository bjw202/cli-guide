튜토리얼 · T10 · 태그

'이 사진이 v1.0이야' — 표시 붙이기

사진첩의 수많은 사진 중 '바로 이 장!'을 한눈에 찾을 수 있게 이름표를 붙이는 방법을 배워요. 그 이름표가 바로 태그랍니다.

이렇게도 써요

음악가는 데모곡을 v1, v2로 나눠 두고, 연구자는 논문 제출 버전을 'submit-v1'으로 딱 찍어둬요. 나중에 '그때 낸 버전'을 정확히 찾아야 할 때 태그가 빛을 발하죠.

음악가 데모 v1 사례 보기 → 논문 제출 버전 사례 보기 →
비유

태그는 사진첩의 한 페이지에 '책갈피 + 제목'을 붙이는 거예요. '아, v1.0이 어디 있더라?' 할 때 40글자 지문을 뒤질 필요 없이 이름만 부르면 그 페이지로 바로 가요.

태그(tag)가 뭔가요?

사진 한 장(커밋)에 '이건 1.0 배포판이야'라고 이름표를 붙이는 걸 태그(tag)라고 해요. 브랜치가 '점점 앞으로 가는 화살표'라면, 태그는 '특정 사진에 영구히 박힐 못'이에요. 움직이지 않고 그 사진을 평생 가리켜요.

태그 두 종류: 가벼운 이름표 vs 정식 이름표

lightweight 태그는 그냥 이름표만 붙이는 거예요. git tag v1.0. 빠르고 가벼워서 '아 여기 기억해둬' 정도로 쓸 때 좋아요. 대신 만든 사람이나 날짜 같은 정보가 안 들어가요.

annotated 태그는 메시지, 만든 사람, 날짜까지 포함된 '정식 이름표'예요. git tag -a v1.0 -m "첫 정식 출시". 공식 배포 버전에는 annotated를 쓰는 게 좋아요. 나중에 '이 버전 누가 언제 만들었지?'를 확인할 수 있거든요.

알아두면 좋은

Semantic Versioning(SemVer): v1.2.3처럼 주.부.수 세 숫자로 의미를 나타내는 약속이에요. 주(major)는 호환이 깨지는 큰 변화, 부(minor)는 호환되는 새 기능, 수(patch)는 버그 수정. 숫자만 봐도 '업데이트해도 안전할까?'를 알 수 있어요.

태그도 구름으로 올려요

태그를 붙였다고 원격에 자동으로 안 올라가요. 따로 올려줘야 해요. 하나만 올릴 땐 git push origin v1.0, 한꺼번에 다 올릴 땐 git push --tags예요. GitHub에 올라간 태그는 'Releases' 페이지에서 보기 좋게 정리돼요.

팁

git tag만 치면 붙인 태그 목록이, git show v1.0을 치면 그 태그가 가리키는 사진의 내용과 메시지가 나와요. 이전 태그를 지우고 싶으면 git tag -d v1.0.

T10 한 장 요약

태그 핵심 명령

git tag v1.0lightweight 태그. 가벼운 이름표만.
git tag -a v1.0 -m "메시지"annotated 태그. 메시지·작성자·날짜 포함.
git tag태그 목록 보기.
git show v1.0해당 태그가 가리키는 사진 내용 보기.
git push origin v1.0태그 하나 원격으로 올리기.
git push --tags모든 태그 한꺼번에 원격으로 올리기.
git tag -d v1.0로컬에서 태그 삭제.
