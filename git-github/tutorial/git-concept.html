튜토리얼 · T01 · Git 기초

시간을 되돌릴 수 있는 사진첩 — 버전 관리란

문서를 고칠 때마다 '최종', '진짜최종', '진짜진짜최종' 파일을 만드셨나요? 버전 관리는 그 짓을 안 해도 되게 해 주는, 시간 여행 사진첩이에요.

이렇게도 써요

소설가도 원고 초안·수정·완결을 버전별로 보관하고, 리눅스 커널 개발자들도 같은 원리로 수만 명의 변경을 한데 모아요. 버전 관리는 글쓰기부터 운영체제 개발까지 쓰이는 같은 도구랍니다.

소설가·작가 사례 보기 → 리눅스 커널 사례 보기 →

먼저, 사진첩으로 시작할게요

컴퓨터 파일이 '책의 한 페이지'라 상상해 보세요. 수정할 때마다 그 페이지를 통째로 복사해서 사진첩에 한 장씩 붙이는 거예요. 언제든 과거 페이지로 돌아갈 수 있고, 두 페이지를 나란히 비교할 수도 있어요. 이 사진첩이 바로 버전 관리 시스템(Version Control System, VCS)이에요.

연결해서 외워요. 사진첩 전체를 저장소(repository)라 부르고, 사진 한 장을 커밋(commit)이라 불러요.

비유

옛날엔 '최종_진짜최종_v3_이게마지막.docx'처럼 파일 이름에 버전을 적었죠. 버전 관리는 그게 필요 없어요. 컴퓨터가 알아서 사진첩에 정리해 주니까요.

한 곳에 모으냐, 여러 군데에 나누냐

사진첨의 생김새는 두 가지예요. 중앙 집중식은 사진첩이 딱 하나뿐이라 모두가 그 한 곳을 오가며 사진을 붙이는 방식이고, 분산식은 참여자마다 사진첩 사본을 통째로 갖는 방식이에요. Git은 분산식이에요 — 덕분에 인터넷이 끊겨도 내 사진첩에 사진을 붙일 수 있고, 나중에 서로 사본을 맞춰요.

옛날 이야기

Git이 있기 전에는 CVS, SVN 같은 '중앙 집중식' 사진첨이 먼저였어요. 2005년, 리눅스를 만든 Linus Torvalds가 '더 빠르고 분산된 사진첩이 필요하다'며 10일 만에 만든 게 Git의 시작이에요.

Git은 '달라진 점'이 아니라 '그 순간 전체'를 찍어요

오래된 시스템은 파일에서 '무엇이 달라졌는지(diff)'만 적어요. Git은 달라요. 사진을 찍을 때 그 순간의 파일 전체 모습(스냅샷)을 저장해요. 큰 그림에서는 스냅샷이 더 빠르고, 과거 어느 시점이든 통째로 꺼내보기 쉬워요.

그리고 각 사진에는 SHA-1이라는 40글자 지문이 찍혀요. 내용이 조금이라도 다르면 지문이 완전히 달라져서, 같은 사진인지 한눈에 알 수 있어요.

한 번 붙인 사진은 바뀌지 않아요

사진첩에 붙인 사진은 불변(immutable)이에요. 고칠 수 없다는 뜻 같지만, 사실 '안전하다'는 뜻이에요. 잘못 붙여도 원래 모습이 어딘가에 온전히 남아 있어서 언제든 되돌아갈 수 있거든요. '되돌리기'도 사진을 지우는 게 아니라 '새 사진을 붙여 반대 방향으로 가는' 거예요.

이때 HEAD라는 표시가 '지금 내가 서 있는 사진'을 가리켜요. HEAD는 '나 여기 있어'라는 깃발이랍니다. 뒤에서 브랜치(T06)를 배울 때 다시 만날 거예요.

알아두면 좋은

Git은 '이상한 영어 단어'가 아니에요. 영국식 발음으로 '깃'인데, 원래 '바보 같은'이라는 뜻의 속어에서 왔대요. Linus의 유머랍니다.

잠깐, Git에는 방이 3개 있다고만 알아두세요

지금은 외울 필요 없어요. 그냥 'Git이 파일을 작업 방·대기 방·기록 방 세 군데로 나눠 본다'는 것만 머릿속에 두세요. 자세한 건 T04 3개의 방에서 다뤄요. 이 세 방을 오가는 흐름이 앞으로 배울 모든 명령의 뼈대랍니다.
