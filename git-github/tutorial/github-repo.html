GitHub에 나의 첫 사진첩 만들기

마을(GitHub)에 도착했으니, 이제 내 집을 지을 차례예요. 그 집의 이름이 바로 저장소(repository)랍니다.

이렇게도 써요

오픈소스에 기여할 땐 남의 저장소를 fork해서, 내 일기를 기록할 땐 private 저장소로 만들어요. 같은 저장소라도 공개·비공개, 원본·사본에 따라 쓰임새가 달라진답니다.

저장소 = 마을에 지은 내 집

GitHub에 올리는 사진첩 하나를 저장소(repository)라고 불러요. 줄여서 'repo(레포)'라고도 해요. T01에서 배운 저장소와 같은 말이지만, 이번엔 그 저장소가 인터넷 마을 위에 올라와 있다는 게 차이예요.

집을 짓는 방법은 여러 가지예요. 빈 땅에 새로 지을 수도 있고(create), 남이 지은 집 도면을 가져와 내 컴퓨터에 똑같이 지을 수도 있고(clone), 남의 집을 통째로 내 이름으로 복사할 수도 있어요(fork). 하나씩 만나봐요.

비유

저장소는 마을에 지은 '집'. create는 빈 땅에 새 집 짓기, clone은 남의 집 도면을 내 작업장에 옮겨오기, fork는 남의 집을 통째로 내 마당에 베껴 짓기예요. 원본 집에 손대지 않고 내 마당에서 마음대로 고칠 수 있죠.

create: 빈 집 짓기

가장 기본. GitHub 웹사이트 우측 위 '+' 버튼 → New repository. 이름, 설명, 공개 여부(public/private)를 정하고 'Create repository'를 누르면 끝이에요. 30초면 돼요.

명령줄을 좋아한다면 GitHub CLI의 gh repo create 한 줄로도 만들 수 있어요. 터미널에서 곧바로 마을에 집을 지을 수 있죠.

clone: 남의 집 도면을 내 작업장으로

이미 GitHub에 있는 저장소를 내 컴퓨터로 복사해 오는 명령이 clone이에요. git clone <주소> 한 줄이면 끝나요. 복사본은 원본과 같은 역사(사진첩)를 통째로 갖고 있어요.

팀 프로젝트라면 보통 이렇게 해요. 한 사람이 만든 저장소를 팀원 모두가 clone해서 각자 컴퓨터에서 작업하고, 나중에 변경을 다시 마을로 올리는(push) 거죠.

fork: 남의 집을 내 마당으로 통째로

fork는 clone과 헷갈리기 쉬워요. 둘 다 '복사'지만, 복사해 오는 곳이 달라요. clone은 남의 저장소를 '내 컴퓨터로' 복사하는 거고, fork는 남의 저장소를 '내 GitHub 계정으로' 복사하는 거예요.

비유하자면, fork는 "남의 레시피북을 통째로 내 책장으로 베껴오기"예요. 원본에 영향을 주지 않고 마음대로 고쳐볼 수 있어요. 고친 걸 원작자에게 제안하고 싶으면 그때 pull request(T16)를 보내는 거예요. 오픈소스 기여의 첫 단계가 바로 fork랍니다.

팁

clone과 fork의 차이를 한 줄로: clone은 '내 컴퓨터로', fork는 '내 GitHub 계정으로'. 오픈소스에 기여하려면 먼저 fork하고, 그 fork를 내 컴퓨터로 clone하는 순서예요.

template: 도면을 재사용하기

비슷한 저장소를 반복해서 만들 일이 있을 때 쓰는 기능이 template repository예요. 자주 쓰는 폴더 구조, 설정 파일, README 틀을 저장소 하나에 템플릿으로 만들어두면, 그걸 시작점 삼아 새 저장소를 찍어낼 수 있어요.

예를 들어 매주 새로 만드는 과제 저장소, 강의 자료 저장소 같은 게 있다면 템플릿 하나 만들어두면 편해요. 'Use this template' 버튼 한 번이면 새 저장소가 템플릿 모양 그대로 태어난답니다.

import: 다른 마을에서 이사 오기

이미 다른 마을(GitLab, Bitbucket)에 집이 있다면, 그걸 GitHub로 그대로 가져올 수 있어요. 이걸 import라고 해요. GitHub의 'Import repository' 메뉴에서 기존 저장소 주소만 넣으면, 역사(사진첩)까지 통째로 이사와 줘요.

이사하면 Git의 사진첩 내용은 그대로지만, 마을의 기능(Issue, PR, Actions)은 GitHub 버전으로 새로 시작해요. 다른 마을에서 깃털 하나 안 떨어지고 이사할 수 있는 셈이죠.

알아두면 좋은

public 저장소는 누구나 볼 수 있고, private 저장소는 내가 초대한 사람만 볼 수 있어요. 비밀 일기, 회사 코드는 private로. 포트폴리오, 오픈소스는 public로. 나중에 마음 바뀌면 설정에서 공개 여부를 바꿀 수도 있어요.

다섯 가지 방법, 정리

저장소를 만드는 다섯 길을 한눈에요.

  • create — 빈 땅에 새 집 짓기 (GitHub New 또는 gh repo create)
  • clone — 남의 저장소를 내 컴퓨터로 복사 (git clone)
  • fork — 남의 저장소를 내 GitHub 계정으로 통째로 복사
  • template — 템플릿 저장소에서 새 저장소 찍어내기
  • import — 다른 마을의 저장소를 GitHub로 이사

집을 지었으니, 이제 마을 사람들과 대화할 차례예요. T16에서 Issue와 Pull Request라는 '대화 도구'를 배워요.
