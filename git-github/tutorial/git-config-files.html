튜토리얼 · T13 · 설정과 파일

Git, 나에게 맞게 조립하기 — 설정과 파일들

Git은 설정 파일, 무시 파일, 훅, 대용량 파일 처리로 입맛대로 조립할 수 있어요. 어떤 파일이 어떤 역할을 하는지 가볍게 익혀둡시다.

이렇게도 써요

개발자는 .gitignore와 aliases로 자기 환경을 dotfiles로 관리하고, GitBook 같은 문서 도구는 Git 설정과 무시 파일을 기반으로 책을 만들어요. 작은 설정 파일 하나가 협업 효율을 크게 바꿔요.

dotfiles 사례 보기 → GitBook 사례 보기 →
비유

Git 설정 파일들은 '내 주방 운영 수칙' 같아요. '이 재료는 냉장고에 넣지 마(.gitignore)', '이 재료는 이렇게 보관해(.gitattributes)', '요리 시작 전 손 씻기(hook)'. 한 번 정해두면 매번 알아서 지켜져요.

.gitignore — '이 파일들은 사진첩에 넣지 마!'

.gitignore는 '이 파일은 사진첩에 넣지 마'라는 무시 목록이에요. 비밀번호 파일, 빌드 결과물, 임시 파일 등을 적어두면 Git이 알아서 무시해요. 아래는 흔히 쓰는 예시예요.

# 비밀 정보 — 절대 올리면 안 됨
.env
.env.local

# 의존성 폴더 (다시 받으면 됨)
node_modules/

# 빌드 결과물
build/
dist/

# 로그 파일
*.log

# OS 쓰레기 파일
.DS_Store
Thumbs.db
조심하세요

.env처럼 비밀이 든 파일은 반드시 .gitignore에! 실수로 push하면 비밀이 공개돼요. 이미 올렸다면 역사에서 지우고(T12 filter-repo), 비밀번호를 바꿔야 해요. (T19 보안과 연계)

팁

github.com/github/gitignore에 흔한 언어·프레임워크별 .gitignore 템플릿이 있어요. Python, Node, macOS 등 골라서 복사하면 시작이 빨라요.

.gitattributes — 파일 성질 알려주기

.gitattributes는 파일의 '성질'을 Git에게 알려줘요. 줄바꿈 처리(윈도우/맥 차이), 바이너리 판별, 병합 방법 등을 정해요. 여러 운영체제에서 같은 사진첩을 쓸 때 줄바꿈이 꼬이는 걸 막아줘요.

hooks — 커밋 전에 자동으로 할 일

hooks(client-side)는 커밋 전, 푸시 전에 자동으로 실행되는 작은 스크립트예요. 예를 들어 '커밋하기 전에 문법 검사 돌려라'를 걸어두면 실수를 미리 막아줘요. .git/hooks/ 폴더에 넣어요. 아래는 커밋 전(pre-commit) 스크립트 예시.

#!/bin/sh
# .git/hooks/pre-commit
# 커밋 전에 간단한 검사 실행
echo "커밋 전 검사 중..."
npm test
if [ $? -ne 0 ]; then
  echo "테스트 실패 — 커밋 중단"
  exit 1
fi

Git LFS & aliases — 큰 파일과 짧은 명령

Git LFS(Large File Storage)는 큰 파일(이미지, 영상, 모델)을 특별 저장소에 따로 보관해요. 사진첩이 무거워지는 걸 막아줘요.

aliases는 긴 명령을 짧게 줄여 쓰는 거예요. git config --global alias.l "log --oneline" 한 번이면 앞으로 git l만 쳐도 긴 로그가 한 줄씩 나와요.

알아두면 좋은

aliases는 --global을 붙이면 그 컴퓨터의 모든 사진첩에, 안 붙이면 현재 사진첩에만 적용돼요. 자주 쓰는 명령은 글로벌로, 프로젝트 전용은 로컬로. 깔끔하게 나눠 쓰세요.

T13 한 장 요약

설정 파일 핵심

.gitignore'이 파일은 사진첩에 넣지 마' 무시 목록.
.gitattributes줄바꿈·바이너리 등 파일 성질 지정.
.git/hooks/커밋/푸시 전 자동 실행 스크립트.
Git LFS큰 파일(이미지·영상) 특별 저장소 보관.
git config alias.l "log --oneline"긴 명령 짧은 별명으로.
git config --global컴퓨터 전체 설정. 생략하면 현재 사진첩만.
