튜토리얼 · T05 · Git 심화

히스토리, 읽는 법과 탐정하는 법

사진첩에 사진이 많이 쌓였어요. 어떻게 읽나요? 이 페이지에서는 사진첩을 넘기는 법(log), 사진 속 한 줄에 화살표를 짚는 법(blame), 그리고 숨겨진 발자국까지 더듬는 법(reflog)을 배워요.

이렇게도 써요

법률가도 계약서의 한 문장이 "누가, 언제, 왜 넣었는지"를 추적해요. Git의 blame은 바로 그 추적 도구예요. 번역가도 같은 문장을 여러 버전으로 다듬을 때, 어느 시점에 어떤 단어를 골랐는지 사진첩을 넘기듯 확인하죠.

법률가 사례 보기 → 번역가 사례 보기 →

사진첩을 넘기는 법 — git log 변형

T03에서 git log로 사진 목록을 봤어요. 사진이 많아지면, 어떻게 보느냐가 중요해져요. 자주 쓰는 네 가지 모양:

  • git log --oneline — 사진 한 장을 한 줄로. 번호(SHA)와 메시지만 딱. 가장 많이 쓰는 모양이에요.
  • git log --graph --all --oneline — 브랜치(T06) 흐름을 그래프로. 사진첩이 어디서 갈라지고 합쳐졌는지 한눈에 들어와요.
  • git log -p — 각 사진의 diff까지 전부. "이 사진에서 정확히 뭐가 바뀌었나"까지 보고 싶을 때.
  • git log --stat — 파일별 통계. "어느 파일이 얼마나 바뀌었나"를 요약으로.

이 옵션들은 섞어 쓸 수 있어요. 예를 들어 git log --graph --oneline --stat처럼. 자기에게 편한 조합을 찾으면 된답니다.

옛날 이야기

git log --graph는 Git 2.0 무렵부터 기본 그래프 옵션으로 자리잡았어요. 그 전엔 ASCII 아트로 직접 그리거나 외부 도구(gitk 등)를 써야 했죠. 요즘은 터미널에서도 브랜치 흐름이 보기 좋게 그려져요.

비유

log는 사진첩을 넘기는 것, blame은 사진 속 인물에게 화살표 짚는 것, reflog는 '내가 사진첩을 언제 어디로 옮겼나'라는 숨겨진 발자국을 더듬는 것이에요.

사진 속 한 줄에 화살표 짚기 — git blame

파일을 열었는데 이상한 한 줄이 있어요. "이 줄은 누가, 언제, 왜 넣었지?"가 궁금할 때가 있죠. 그때 쓰는 도구가 git blame이에요. 이름은 좀 무서워 보이지만, 실제론 '책임 소재를 찾는다'는 뜻이 아니라 '이 줄의 출처를 찾는다'는 탐정 도구예요.

git blame readme.md

그러면 파일의 모든 줄 옆에 그 줄을 마지막으로 바꾼 사진의 번호, 바꾼 사람, 날짜가 붙어서 나와요. 이상한 줄을 발견하면 그 옆의 사진 번호를 보고 git show <번호>로 그 사진을 확대하면, "왜 이 줄이 생겼는지" 맥락까지 알 수 있어요.

"남 탓"하려는 게 아니에요. 같이 일할 때 '이 부분은 왜 이렇게 됐지?'를 물어보려면 출처를 알아야 하니까요. blame은 그 출처를 가르쳐 주는 도구예요.

숨겨진 발자국 — git reflog

git log는 사진첩에 붙인 사진들을 보여줘요. 하지만 Git은 그보다 더 자세한 기록을 남겨요 — 내가 깃발(HEAD)을 어디로 옮겼는지 매번 기록해요. 이 숨겨진 발자국을 보는 명령이 git reflog예요.

git reflog

reflog는 사진첩 넘기는 책갈피 같아요. "아까 봤던 사진으로 다시 가고 싶어" 같은 이동, 심지어 실수로 사진을 버린 경우까지 모두 여기에 발자국이 남아요. 그래서 실수로 잃어버린 사진도 reflog에서 찾아 되돌릴 수 있어요.

이건 T11(undo/되돌리기)과 직결되는 이야기예요. "Git에서는 진짜로 뭘 지워버리기 전엔 잃어버린 게 거의 없다"는 말은, 결국 이 reflog 덕분이에요. 당황하지 말고 발자국을 더듬으면 돼요.

특정 사진 확대하기 — git show

T03에서 잠깐 봤던 git show를 다시 정리할게요. git show만 치면 가장 최근 사진을, git show <번호>처럼 사진 번호(SHA)를 주면 그 사진을 확대해요. 무엇이 바뀌었는지(diff)까지 같이 보여줘요.

흐름은 이래요. git log --oneline으로 목차를 보고 → 궁금한 사진 번호를 고르고 → git show <번호>로 그 사진을 확대하고 → 더 파고 싶으면 git blame으로 줄 단위까지. 이 네 명령(log, show, blame, reflog)이 있으면 사진첩의 어느 구석이든 탐험할 수 있어요.

알아두면 좋은

사진 번호(SHA)를 전부 40글자 칠 필요 없어요. 앞 7글자만 써도 Git이 알아서 알아들어요. a1b2c3d처럼 7글자면 보통 충분하고, 아주 큰 사진첨에선 겹칠 수 있으니 그때는 몇 글자 더 붙이면 된답니다.

T05 한 장 요약

히스토리 읽기·탐정하기 명령

git log --oneline사진을 한 줄 요약으로. 가장 많이 쓰는 모양.
git log --graph --all브랜치 흐름을 그래프로. 갈라짐/합쳐짐 한눈에.
git log -p각 사진의 diff까지 전부 표시.
git log --stat파일별 변경 통계 요약.
git blame <파일>각 줄의 출처(누가·언제·어느 사진에서) 추적.
git reflogHEAD 이동 발자국. 잃어버린 사진 복구 단서 (T11 연계).
git show [<번호>]특정 사진 확대. diff까지. 기본=최근 사진.
