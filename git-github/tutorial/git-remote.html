튜토리얼 · T09 · 원격 저장소

내 사진첩, 구름 위로 올리기 — 원격 저장소

지금까지는 내 컴퓨터 안에서만 사진첩을 봤어요. 이제 그 사진첩을 GitHub라는 '구름 위의 사진첩'에 올려서 다른 컴퓨터, 다른 사람과 나눠 보겠어요.

이렇게도 써요

개발자들은 집 컴퓨터와 회사 컴퓨터의 설정 파일(dotfiles)을 GitHub에 올려두고 어디서든 자기 환경을 그대로 불러와요. 원격 저장소가 있어서 가능한 일이에요.

dotfiles — 여러 컴퓨터 설정 동기화 사례 보기 →
비유

원격 저장소는 '구름 위의 사진첩'이에요. git push는 사진을 구름으로 올리는 거고, git pull은 구름에서 사진을 받아오는 거예요. 같은 사진첩을 여러 기기에서 동기화할 수 있죠.

원격 저장소(remote)가 뭔가요?

지금까지 우리 사진첩은 '내 컴퓨터'라는 서랍 안에만 있었어요. 이제 그 사진첩을 인터넷 너머의 다른 컴퓨터로 보내서, 다른 컴퓨터도 같은 사진첩을 갖게 만들 수 있어요. 그 '다른 컴퓨터 쪽의 사진첩'을 원격 저장소(remote repository)라고 불러요.

가장 유명한 원격 저장소 서비스가 바로 GitHub예요. 그 외에도 GitLab, Bitbucket 같은 친구들이 있어요. Git은 원격 저장소를 가리키는 짧은 이름을 붙이는데, 관례상 첫 원격은 origin이라고 불러요.

조심하세요

git push --force는 원격의 역사를 '내 것'으로 덮어써요. 동료가 올려둔 사진이 통째로 사라질 수 있어요. 정말 필요할 때만, 그리고 안전장치가 있는 git push --force-with-lease를 대신 쓰세요.

연결하고, 올리고, 받고 — 네 개의 기본 동작

원격과 주고받는 동작은 크게 네 가지예요. 먼저 연결하고 (git remote add origin <주소>), 내 사진을 올리고 (git push), 상대 사진을 받아오고 (git fetch / git pull), 통째로 복사해 오고 (git clone). 아래 터미널을 스크롤하면 한 번에 흐름이 보여요.

알아두면 좋은

fetch는 '엿보기', pull은 '엿보고 합치기'예요. pull이 편하지만, 변경을 먼저 살펴보고 합칠지 결정하고 싶을 땐 fetch로 먼저 보세요. pull은 사실 fetch + merge 두 단계를 한 번에 하는 거랍니다.

받아올 때 '병합' 대신 '리베이스'로

git pull은 기본적으로 받아온 사진을 병합(merge) 방식으로 합쳐요. 그러면 '병합했습니다'라는 사진이 하나 더 생겨요. 역사를 깔끔하게 한 줄로 유지하고 싶다면 git pull --rebase를 써 보세요. 받아온 사진 위에 내 사진을 다시 올려놓는(rebase) 방식이라 병합 사진이 안 생겨요. T08에서 배운 리베이스와 같은 원리예요.

처음부터 통째로 가져오기: clone

내 컴퓨터에 아무것도 없고 원격 사진첩만 있다면, git clone <주소> 한 번이면 원격 사진첩 전체를 내 컴퓨터로 통째로 복사해 와요. 연결(origin)까지 자동으로 잡혀요. 남이 공개해 둔 프로젝트를 처음 가져올 때 쓰는 첫 명령이에요.

T09 한 장 요약

원격 저장소 핵심 명령

git remote add origin <주소>원격 사진첩을 origin이라는 이름으로 연결.
git fetch원격의 변경을 '가져와서 보기만' (적용 안 함).
git pullfetch + merge. 가져와서 합치기.
git pull --rebase병합 대신 리베이스로 합치기. 역사 한 줄 유지.
git push내 사진을 원격으로 올리기.
git push --force-with-lease강제 push의 안전 버전. 동료 작업 보호.
git clone <주소>원격 사진첩을 통째로 복사해 오기.
