한 권의 책을 여러 책상에 펼쳐놓기 — git worktree

T12에서 잠깐 소개한 worktree를 더 깊이 파요. 브랜치를 여러 개 동시에 작업하고 싶을 때, 파일을 전환하는 대신 폴더를 전환하는 방법이에요.

이렇게도 써요

한 저장소를 여러 책상에 펼쳐 각각 다른 브랜치를 동시에 작업하는 건, 거대 오픈소스 팀이 여러 기능을 병렬로 개발할 때 흔히 쓰는 방식이에요.

모노레포 대규모 관리 사례 보기 →
비유

보통은 책(저장소)을 한 책상에서만 펼쳐봐요. 다른 장(브랜치)을 보려면 페이지를 넘기죠(git switch). worktree는 같은 책을 여러 책상에 동시에 펼쳐놓는 거예요. 각 책상은 각자 다른 장을 펼쳐놓고, 한눈에 전부 볼 수 있어요.

worktree가 뭔가요?

git worktree는 하나의 저장소를 여러 폴더에서 동시에 열어주는 기능이에요. 각 폴더는 각자 다른 브랜치를 체크아웃해요. 핵심은 — 이 폴더들은 같은 사진첩(.git)을 공유한다는 거예요. 그래서 사진첩을 두 번 복사하지 않고도, 여러 브랜치를 폴더처럼 나란히 둘 수 있어요.

보통 브랜치를 바꿀 땐 git switch로 작업 폴더가 통째로 덮어씌워져요. 한 브랜치에서 빌드 중인데 다른 브랜치를 봐야 하면 곤란하죠. worktree는 그때 "다른 책상에 앉아서 다른 장을 본다"는 해결책이에요.

언제 쓰나요?

  • 긴 작업을 두고 긴급 수정이 들어올 때 — 메인 책상은 그대로 둔 채, 옆 책상에서 hotfix 브랜치를 열어 고치고 합친 뒤 닫아요. 진행 중이던 작업을 엎지 않아도 돼요.
  • 두 브랜치를 비교하며 작업할 때 — 예전 버전과 지금 버전을 나란히 띄워놓고 코드를 비교할 수 있어요.
  • 오래 걸리는 빌드/인덱스를 유지하고 싶을 때 — 브랜치를 바꾸면 에디터가 파일을 다시 읽고 인덱스를 다시 만들어요. worktree는 폴더마다 인덱스가 독립이라, "이 책상은 계속 켜둔 채로"가 가능해요.
  • 여러 기능을 동시에(병렬로) 개발할 때 — 특히 AI 에이전트나 동료와 각자 worktree에서 다른 브랜치를 동시에 코딩하는 패턴에 잘 맞아요(아래 '에이전틱 코딩' 절 참고).

실전 명령어 — 책상 하나 더 꺼내기

메인 폴더(예: cli-guide/)에서 시작한다고 해요. 거기가 첫 번째 worktree(main 브랜치)예요.

# ① 새 책상(worktree) 추가 — 새 브랜치로
git worktree add ../cli-guide-feature  feature/login

# ② 그 책상으로 가서 작업
cd ../cli-guide-feature
#   ...여기선 feature/login 브랜치가 열려 있어요.
#   메인 책상(cli-guide/)은 여전히 main 브랜치 그대로!

# ③ 작업 끝나면 메인으로 돌아와 합치기
cd ../cli-guide
git merge feature/login

# ④ 다 쓴 책상 치우기
git worktree remove ../cli-guide-feature
git branch -d feature/login   # 브랜치도 정리

다른 책상에서 한 작업은 같은 사진첩에 바로 기록돼요. 별도로 push하거나 복사할 필요 없어요 — worktree는 같은 .git를 공유하거든요.

책상 목록 보기 & 잃어버린 책상 치우기

git worktree list     # 현재 열려 있는 책상 전체 보기
git worktree prune    # 폴더를 그냥 지워버렸을 때 썩은 기록 정리

list로 "내가 책상을 몇 개나 펼쳐뒀더라?"를 확인하고, prune은 폴더를 remove 안 하고 그냥 삭제했을 때 남는 찌꺼기를 치워요.

에이전틱 코딩 — 여러 AI가 책상마다 앉아서

최근 많이 쓰이는 패턴이에요. 기능을 여러 개로 쪼개서, 각 worktree에서 AI 에이전트(또는 동료)가 동시에 코딩하게 한 뒤, 메인에서 합치는 방식이에요.

# 기능별 책상을 미리 깔기
git worktree add ../proj-cases   feature/cases
git worktree add ../proj-assets  feature/assets
git worktree add ../proj-qa      feature/qa

# 각 폴더에서 에이전트를 따로 띄워 동시에 작업
cd ../proj-cases  &&  claude   # 사례 갤러리 담당
cd ../proj-assets &&  claude   # 인터랙티브 자산 담당 (동시 진행)

# 다 되면 메인에서 합치고 책상 정리
cd ../proj
git merge feature/cases feature/assets

Claude Code처럼 에이전트를 다루는 도구는 worktree 격리를 직접 지원해요. claude --worktree(또는 -w)로 격리된 세션을 바로 열 수 있고, 에이전트를 띄울 때 자동으로 전용 worktree에 넣어주는 옵션도 있어요. 핵심 장점: 한 에이전트가 실험하다 main을 더럽혀도, 다른 에이전트의 책상은 멀쩡해요.

팁

기능이 서로 독립적일수록(같은 파일을 안 건드릴수록) 병렬 worktree가 빛을 발해요. 같은 파일을 동시에 고치면 결국 병합 충돌이 나거든요. "각자 다른 폴더/파일 담당"이 병렬 개발의 전제예요.

조심하세요

worktree를 다 썼으면 git worktree remove로 정리하세요. 폴더를 그냥 삭제하면 git이 "책상이 아직 열려 있다"고 착각해요(이때 prune으로 치우면 됨). 그리고 같은 브랜치를 두 worktree에서 동시에 열 수는 없어요 — 한 브랜치는 한 책상에서만.

알아두면 좋은

worktree는 clone과 달라요. clone은 사진첩을 통째로 복사하는 거고(별개 저장소), worktree는 같은 사진첩을 다른 폴더에서 열어보는 거예요. 그래서 worktree끼리는 커밋이 즉시 공유돼요.

T22 한 장 요약

git worktree 핵심 명령

git worktree add <경로> <브랜치>새 책상(worktree) 추가. 그 폴더에 해당 브랜치가 열림.
git worktree list현재 열린 worktree 전체 보기.
git worktree remove <경로>다 쓴 worktree 정리. (폴더 그냥 지우지 말 것)
git worktree prune폴더를 그냥 지운 뒤 남은 찌꺼기 정리.
claude --worktree (-w)격리된 worktree 세션에서 Claude Code 실행.
