튜토리얼 · T12 · 고급 기법

필요할 때 찾게 되는 고급 도구들

이 도구들은 매일 안 써도 '아, 그때 이걸 알았으면!' 하는 순간이 반드시 와요. 지금은 가볍게 훑어만 보세요. 필요해질 때 다시 돌아오면 돼요.

이렇게도 써요

연구자는 논문에 버그가 생긴 시점을 bisect로 이진 탐색하고, 거대 프로젝트는 submodule로 여러 저장소를 한 곳에 끼워 넣어 관리해요. '고급'이지만 현장에서는 일상 도구예요.

비유

고급 도구들은 주방의 특수 도구 같아요. 매일 쓰는 칼·프라이팬은 아니지만, '이탈리아 면 만들 때', '케이크 장식할 때' 딱 필요한 순간이 와요. 이름과 용도만 알아도 충분해요.

cherry-pick — 체리 한 알만 따오기

git cherry-pick <커밋>는 다른 브랜치의 사진 중 딱 하나만 내 브랜치로 가져와요. 비유하자면 나무에서 체리 한 알만 골라 따는 거예요. '이 기능만 이 브랜치에도 넣고 싶다' 할 때 유용해요.

stash — 작업을 잠시 서랍에

git stash는 작업 중인 변경을 잠시 서랍에 넣어요. '방 청소하느라 물건 잠시 서랍에'처럼요. 브랜치를 바꿔야 하는데 아직 커밋하기엔 이를 때 쓰고, 돌아와서 git stash pop으로 다시 꺼내요.

알아두면 좋은

stash는 '잠깐 비켜' 명령어예요. 브랜치를 바꿔야 하는데 커밋하기엔 이른 변경이 있을 때, stash로 잠시 치워두고 나중에 다시 꺼내 쓰세요. 커밋보다 가볍고, 임시 보관 느낌이에요.

bisect — 반씩 쪼개며 범인 찾기

어느 날부터 버그가 생겼는지 모를 때 git bisect가 빛을 발해요. '좋은 사진'과 '나쁜 사진'을 알려주면, Git이 사진첩을 반씩 쪼개가며 '범인(처음 버그가 생긴 사진)'을 이진 탐색으로 찾아줘요. 수백 장의 사진 속에서도 10번 안에 찾아요.

submodule & worktree — 사진첩 안의 사진첩, 여러 책상

git submodule은 한 사진첩 안에 다른 사진첩을 끼워 넣는 거예요. '책 안에 다른 책을 부록으로' 넣는 것처럼요. 큰 프로젝트가 여러 저장소를 모아 쓸 때 쓰여요.

git worktree는 같은 사진첩을 여러 폴더에서 동시에 열어요. '같은 책을 여러 책상에 펴놓기'예요. 브랜치 여러 개를 동시에 봐야 할 때 폴더를 바꾸는 번거로움을 줄여줘요.

git filter-repo — 역사 전체에서 한 사람 모자이크

git filter-repo는 사진첩 전체에서 특정 파일이나 비밀번호를 제거해요. 모든 사진을 뒤져 '이 사람 모자이크'처럼 흔적을 지워요. 비밀번호를 실수로 커밋했을 때 꼭 필요해요.

조심하세요

비밀번호를 실수로 커밋했다면 filter-repo로 역사에서 제거해야 해요. 그리고 그 비밀번호를 바꿔야 해요. reset만으로는 원격에 이미 남아있기 때문이에요. 한 번 올라간 비밀은 '없었던 일'로 만들 수 없어요.

고급 기법 핵심 명령

git cherry-pick <커밋>다른 브랜치의 사진 한 장만 따오기.
git stash작업 중 변경을 잠시 서랍에 보관.
git stash pop보관해둔 변경 다시 꺼내기.
git bisect이진 탐색으로 버그 범인 찾기.
git submodule사진첩 안에 다른 사진첩 끼워 넣기.
git worktree같은 사진첩을 여러 폴더에서 열기.
git filter-repo역사 전체에서 파일/비밀번호 제거.
