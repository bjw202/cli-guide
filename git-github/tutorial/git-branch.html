튜토리얼 · T06 · 브랜치·병합

같은 시점에서 갈라지는 또 하나의 세계 — 브랜치

원본을 망가뜨리지 않고 실험하고 싶을 때, Git은 '평행 우주'를 만들어 줘요. 그 우주의 이름을 브랜치(branch)라고 불러요.

이렇게도 써요

디자이너는 '버튼을 파랑으로 할까, 코랄로 할까' 두 안을 브랜치로 나눠 동시에 실험해요. 음악가도 '이 곡을 어쿠스틱 편곡으로, 전자 음악으로' 갈래를 만들어 따로 작업한답니다.

소설을 쓰다가 결말을 둘로 써보고 싶다면

소설을 쓰다가 '결말을 행복하게 할까, 비극적으로 할까' 고민된다고 해요. 원고를 복사해 두 개 파일로 나누면 되지만, 나중에 어느 쪽이 최신인지 헷갈리고 두 파일을 번갈아 고치다 꼬이기 쉽죠. Git은 이 고민을 '평행 우주'로 풀어줘요.

원본 우주(main)는 그대로 두고, 거기서 갈라진 새로운 우주를 하나 만드는 거예요. 이 우주에서는 마음껏 실험해도 원본이 흔들리지 않아요. 실험이 마음에 들면 원본으로 합치면 되고(T07에서 배워요), 아니면 그 우주를 그냥 버리면 돼요. 이 '갈라진 우주'를 브랜치(branch)라고 불러요.

비유

브랜치는 '원본을 손대지 않고 실험하는 복제 방'이에요. 실험이 맘에 들면 원본 방으로 가져오면 되고, 맘에 안 들면 그 복제 방을 그냥 닫아버리면 끝이에요.

평행 우주 만들기, 옮겨가기, 없애기

브랜치를 다루는 기본 명령은 다섯 개예요. 본문에서는 명령어를 알약 모양(git branch)로 표시할게요.

먼저 새 우주를 만들 때는 git branch <이름>이에요. 만들기만 할 뿐 아직 그쪽으로 안 간다는 점에 주의하세요. 그쪽으로 발을 옮기려면 git switch <이름>이 필요해요. 옛날 명령인 git checkout <이름>도 같은 일을 하지만, 요즘은 더 명확한 git switch를 권해요.

만들면서 바로 그쪽으로 가고 싶다면 git switch -c <이름>을 한 번에 쓰면 돼요. 우주가 몇 개 있는지 보려면 그냥 git branch, 지우려면 git branch -d <이름>이에요.

주의

아직 main에 안 합친 브랜치를 git branch -D(대문자 D, 강제 삭제)로 지우면 그 우주의 작업이 날아가요. 실수로 날리는 걸 막으려면 소문자 -d를 쓰세요. 안 합친 게 있으면 Git이 지우지 못하게 막아준답니다.

HEAD: '나 지금 이 우주에 있어'라는 깃발

T01에서 잠깐 봤던 HEAD를 다시 만나요. HEAD는 '지금 내가 발을 딛고 있는 사진(우주)'을 가리키는 깃발이에요. git switch로 브랜치를 옮길 때마다 HEAD가 '이쪽이야' 하고 옮겨 붙어요. 그래서 같은 저장소에서 작업하다가 브랜치를 바꾸면, 파일들이 그 우주의 모습으로 바뀌는 걸 눈으로 볼 수 있어요.

detached HEAD: 브랜치가 아니라 '사진' 위에 서 있을 때

보통 HEAD는 브랜치 이름표를 통해 사진을 가리켜요. 그런데 옛날 특정 사진에 직접 서고 싶을 때가 있어요. 그러면 HEAD가 브랜치에서 떨어져 나온 채로 한 장의 사진을 가리키게 돼요. 이걸 detached HEAD(분리된 HEAD)라고 불러요.

이 상태에서는 '과거를 구경하는 관광'은 안전해요. 다만 여기서 새 작업을 커밋하면 어느 브랜치에도 속하지 않는 떠돌이 사진이 될 수 있어요. 구경만 할 거라면 아무 문제 없지만, 작업은 보통 새 브랜치를 만들어 그 위에서 하세요.

알아두면 좋은

브랜치는 '사진을 복사'하는 게 아니에요. 그저 '이름표 → 사진' 하나의 포인터일 뿐이에요. 그래서 브랜치를 수백 개 만들어도 용량을 거의 안 먹고, 옮겨 다니는 것도 찰나예요. Git이 가벼운 건 이 때문이랍니다.

가지가 뻗어나가는 그래프 상상하기

커밋들이 한 줄로 늘어선 걸 생각해 보세요. 거기서 어느 시점에서 가지를 쳐 옆으로 새 줄이 뻗어나가요. 원본 줄은 계속 앞으로 가고, 갈라진 줄도 각자 앞으로 나아가요. 이 모양이 바로 Git 그래프예요. git log --graph --oneline을 치면 터미널에서 이 가지 모양을 직접 볼 수 있어요.

이 그래프를 한 번 눈에 익히면, 다음 장(T07 병합)에서 두 줄이 다시 하나로 합쳐지는 모습이 자연스럽게 와닿을 거예요.

팁

브랜치 이름은 '무엇을 하려는지' 적는 게 나중에 봤을 때 편해요. feature/login, fix/typo-readme처럼 접두어를 붙이는 팀이 많아요. 띄어쓰기는 안 돼요 — 빈칸 대신 하이폰(-)을 쓰세요.

T06 한 장 요약

브랜치 다루기 기본 명령

git branch <이름>새 평행 우주(브랜치) 만들기. 이동은 별도.
git switch <이름>그 우주로 이동. checkout <이름>과 같은 일.
git switch -c <이름>만들면서 바로 그쪽으로 이동.
git branch브랜치 목록 보기.
git branch -d <이름>브랜치 삭제(-d 안전, -D 강제).
HEAD'지금 내가 서 있는 사진/우주'를 가리키는 깃발.
detached HEAD브랜치가 아닌 특정 사진에 직접 서 있는 상태.
