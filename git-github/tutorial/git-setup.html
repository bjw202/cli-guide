튜토리얼 · T02 · Git 기초

Git, 내 컴퓨터에 초대하기

사진첩을 쓰려면 먼저 컴퓨터에 Git이라는 프로그램을 깔아야 해요. 그리고 '내가 누구인지' 한 번만 알려주면, 그다음부턴 사진마다 내 이름이 자동으로 들어간답니다.

이렇게도 써요

설정을 한 번 해 두면 집 컴퓨터, 회사 컴퓨터, 노트북 어디서나 똑같은 환경에서 일할 수 있어요. 숙련된 사용자들은 이 설정 자체를 Git으로 관리한대요. 이름하여 'dotfiles' 관리.

dotfiles 사례 보기 →

왜 '설치'부터 시작하나요?

앞 페이지(T01)에서 Git은 '시간 여행 사진첩'이라고 했죠. 그 사진첩은 실제로는 컴퓨터에 깔리는 작은 프로그램이에요. 운영체제(Windows, macOS, Linux) 안에는 기본적으로 들어있지 않기 때문에, 우리가 직접 초대해야 해요. 한 번만 설치하면 평생 쓸 수 있어요.

설치는 각 운영체제마다 방식이 달라요. 아래에서 내 컴퓨터에 맞는 방법을 골라 보세요.

운영체제별 설치

macOS (맥)

맥에서는 두 가지 길이 있어요. 가장 쉬운 건 터미널에서 한 줄을 치는 거예요. Homebrew라는 '앱스토어 비슷한 도구'가 이미 있다면:

brew install git

Homebrew가 없다면, 그냥 터미널에 git이라고 쳐보세요. 맥이 알아서 "개발자 도구를 설치할까요?"라고 물어봐요(Xcode Command Line Tools). 동의하면 끝.

Linux (우분투/데비안 계열)

리눅스는 터미널 한 줄이면 됩니다. 리눅스의 '앱스토어 명령어'인 apt를 써요:

sudo apt install git

다른 배포판(Fedora, Arch 등)은 명령어가 달라요. 각 배포판의 패키지 관리자(dnf, pacman 등)로 git을 찾으면 됩니다.

Windows

윈도우는 설치 마법사를 따라가는 게 제일 편해요. Git for Windows 인스톨러를 공식 홈페이지(git-scm.com)에서 내려받아 실행하세요. '다음' 버튼만 누르면 되고, 설치가 끝나면 Git Bash라는 터미널이 생겨요. 앞으로 이 터미널에서 명령어를 칠 거예요.

설치 확인

설치가 끝났으면 터미널에 git --version을 쳐보세요. git version 2.XX.X 같은 줄이 나오면 성공이에요. 숫자는 버전이 바뀌면 달라져요.

비유

git config는 사진첩에 적힌 내 명함이에요. 한 번만 찍어두면, 이후 사진첩에 붙이는 모든 사진에 내 이름이 자동으로 같이 들어가요.

첫 인사 — 내가 누군지 알려주기

Git은 사진을 찍을 때마다 "이 사진은 누가 찍었어?"라는 질문을 해요. 매번 답하기 귀찮으니, 미리 답을 등록해 두는 게 config예요. 딱 두 번만 치면 평생 끝이에요.

git config --global user.name "본인이름"
git config --global user.email "본인이메일@example.com"

--global은 '이 컴퓨터에서 앞으로 하는 모든 프로젝트에 같은 이름/이메일을 써라'라는 뜻이에요. 한 번만 하면 다시 안 해도 돼요.

제대로 들어갔는지 확인하고 싶다면:

git config --global user.name
git config --global user.email

등록한 이름과 이메일이 그대로 출력되면 성공이에요.

잠깐, 주의!

이메일은 공개 저장소(GitHub 등)에 누구나 볼 수 있게 기록될 수 있어요. 이메일을 숨기고 싶다면 GitHub가 제공하는 noreply 이메일(12345+username@users.noreply.github.com 형식)을 쓰는 게 안전해요. GitHub 설정 페이지에서 찾을 수 있어요.

사진첩 두 가지 방식으로 만들기

설치와 인사가 끝났으면, 이제 사진첩을 하나 만들 차례예요. 두 가지 상황이 있어요.

처음부터 새로 만들 때 — git init

내 컴퓨터의 어떤 폴더를 사진첩으로 만들고 싶다면, 그 폴더 안에서:

git init

빈 사진첩이 하나 만들어져요. 폴더 안에 숨겨진 .git이라는 폴더가 생기는데, 이게 사진첩의 실체예요.

남이 만든 사진첩을 복사해 올 때 — git clone

이미 누군가 공유한 사진첩(GitHub에 올라온 프로젝트 등)을 통째로 내 컴퓨터로 가져오고 싶다면:

git clone https://github.com/누군가/프로젝트.git

인터넷에서 사진첩 전체를 한 번에 내려받아요. 복사본은 내 컴퓨터에 새 폴더로 생기고, 그 안에도 .git이 있어요.

둘 다 결과는 같아요 — 내 컴퓨터에 '사진첩'이 생겼다는 것. 다음 페이지(T03)에서는 이 사진첩에 사진을 한 장씩 붙이는 법을 배워요.

알아두면 좋은

.git 폴더는 사진첩의 심장이에요. 이 폴더를 지우면 사진첩이 보통 폴더로 돌아가요. 반대로 .git 폴더만 있으면 어디서든 사진첩 전체가 복원돼요. 그래서 이 폴더를 백업하는 것만으로도 프로젝트 이력 전체가 안전해진답니다.
