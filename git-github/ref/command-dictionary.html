참조 허브 · R01

명령어 사전

"git reset 차이가 뭐였지?" 필요한 순간에 점프. 주제·난이도로 거르고, 각 명령은 튜토리얼로 바로 이어져요. URL fragment(#git-reset)로 직접 링크도 가능합니다.

주제
난이도
git init

빈 사진첩(저장소)을 새로 만드는 첫 명령. 현재 폴더를 Git이 추적하기 시작해요.

기본 초급 → T02 자세히
git clone <url>

원격 저장소 전체를 내 컴퓨터로 복사해 오기. 역사까지 전부 가져와요.

기본 · 원격 초급 → T09 자세히
git config

Git 설정(이름·이메일 등)을 읽고 고치기. 사진첩에 도장 찍을 주인 정보를 정해요.

기본 초급 → T13 자세히
git status

지금 뭐가 바뀌었고, 사진첩에 안 붙인 게 뭔지 보여줘요. 막막하면 일단 status.

기본 초급 → T03 자세히
git add <file>

바뀐 파일을 '다음 사진에 넣겠다'고 표시(대기방으로 올리기). .으로 전부 추가.

기본 초급 → T04 자세히
git commit -m "msg"

대기방의 묶음에 도장 찍어 사진첩에 한 장으로 영구히 붙이기. --amend로 직전 사진 수정.

기본 초급 → T03 자세히
git log

사진첩의 목차. --oneline(한 줄 요약) · --graph --all(그림) · -p(diff 포함).

기본 중급 → T05 자세히
git show <커밋>

사진 한 장을 확대해 보기. 무엇이 바뀌었는지(diff)까지 자세히 보여줘요.

기본 중급 → T05 자세히
git diff

작업방·대기방·사진첩 사이의 차이를 비교. --stat(요약) · 브랜치 간 비교도 가능.

기본 중급 → T04 자세히
git blame <file>

이 줄을 누가, 언제, 어떤 사진에서 남겼는지 한 줄씩 추적. 코드 흔적 수사.

고급 중급 → T05 자세히
git reflog

HEAD가 지나간 모든 흔적(숨은 발자취). reset --hard로 잃어버린 커밋을 찾아 되돌릴 때 구명조끼.

복구 · 고급 고급 → T11 자세히
git branch

평행 우주(브랜치) 목록 보기 · 새로 만들기 · 지우기. git branch <이름>로 생성.

브랜치 중급 → T06 자세히
git switch <브랜치>

다른 평행 우주로 이동. -c로 새 브랜치 만들면서 이동. checkout의 후속 명령.

브랜치 중급 → T06 자세히
git merge <브랜치>

두 평행 우주를 하나로 합치기. 빠른길(fast-forward)과 3-way 병합이 있고, 충돌이 날 수 있어요.

브랜치 중급 → T07 자세히
git rebase <브랜치>

내 사진들을 다른 우주 끝으로 '옮겨 다시 붙이기'. 역사를 한 줄로 깔끔하게 정리. -i로 대화형.

브랜치 · 고급 고급 → T08 자세히
git remote

원격 저장소 연결 관리. git remote add origin <url>로 GitHub와 이어줘요.

원격 중급 → T09 자세히
git fetch

원격의 새 소식을 '받아만' 오기. 내 작업방은 건드리지 않아 안전하게 미리 보기.

원격 중급 → T09 자세히
git pull

fetch + merge 한 번에. 원격 변경을 받아와 바로 합치기. --rebase 옵션도.

원격 중급 → T09 자세히
git push

내 사진첩을 원격으로 올리기. -u로 처음 연결. 공개된 사진은 함부로 고치면 위험(참고: --force-with-lease).

원격 중급 → T09 자세히
git tag

특정 사진에 이름표(버전) 붙이기. lightweight와 annotated(서명 포함) 두 종류.

브랜치 · 고급 중급 → T10 자세히
git reset

과제 되돌리기. --soft(HEAD만) · --mixed(대기방까지, 기본값) · --hard(작업방까지 버림).

복구 고급 → T11 자세히
git revert <커밋>

과거 사진을 '부정'하는 새 사진을 추가. 역사를 안전하게 되돌리며, 이미 push한 공개 내역에 안전해요.

복구 중급 → T11 자세히
git restore <file>

파일을 이전 상태로 되돌려 놓기. 대기방 올린 것 취소(--staged)도 가능. checkout <file> 후속.

복구 · 기본 중급 → T11 자세히
git clean

추적 안 된(새로 만든) 파일을 작업방에서 쓸어내기. -fd로 폴더까지. 복구 불가니 주의.

복구 중급 → T11 자세히
git cherry-pick <커밋>

다른 우주의 특정 사진 한 장만 골라 내 우주로 가져오기. 전체 병합 없이 한 개만.

고급 고급 → T12 자세히
git stash

작업중인 변경을 잠시 서랍에 넣어두기. 급히 브랜치 바꿔야 할 때 유용. pop으로 꺼내기.

고급 중급 → T12 자세히
git bisect

이진 탐색으로 버그가 생긴 사진을 찾기. 좋은/나쁜 사진 표시하면 Git이 반씩 좁혀 범인을 찾아요.

고급 고급 → T12 자세히
git submodule

저장소 안에 다른 저장소를 '하위 모듈'로 끼워 넣기. 공통 라이브러리를 가져다 쓸 때.

고급 고급 → T12 자세히
git worktree

한 저장소를 여러 폴더에서 동시에(각각 다른 브랜치로) 작업. 브랜치 전환 없이 병렬 개발.

고급 고급 → T12 자세히
git filter-repo

역사 전체에서 민감 정보(비밀번호, 큰 파일)를 통째로 씻어내기. 실수로 push한 비밀 제거용.

고급 · 복구 고급 → T12 자세히
.gitignore

추적하지 않을 파일 목록 파일. 빌드 결과·비밀(.env)을 사진첩에서 영원히 배제.

기본 초급 → T13 자세히

GitHub 기능

Fork

남의 저장소를 내 계정로 통째로 복사. 마음대로 실험한 뒤 원작자에게 PR로 제안.

github 중급 → T15 자세히
Issue

대화방에 남기는 '할 일·버그·질문' 카드. 라벨·마일스톤으로 정리. gh issue로 터미널에서도.

github 초급 → T16 자세히
Pull Request (PR)

"내 변경을 받아주세요" 하는 제안 카드. 리뷰→승인→병합(squash/rebase/merge) 흐름. gh pr.

github 중급 → T16 자세히
Actions (workflow.yml)

push 등 이벤트에 자동으로 돌아가는 로봇. job/step/runner/matrix/secret으로 CI·CD 자동화.

github 고급 → T18 자세히
gh (GitHub CLI)

터미널에서 GitHub를 다루는 공식 도구. gh repo · gh pr · gh issue로 브라우저 없이 협업.

github 중급 → T18 자세히

조건에 맞는 명령어가 없어요. 필터나 검색어를 바꿔보세요.
