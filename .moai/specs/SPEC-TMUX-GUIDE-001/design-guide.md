# 디자인 가이드 — 비개발자를 위한 tmux 단일 페이지

> 본 파일은 **디자인/자산 전략 기획** 이다. 실제 CSS 코드/에셋 파일은 구현 Phase에서
> 제작된다. 모든 에셋은 "비개발자 학습 가치" 정당성을 포함한다.

---

## 1. 톤앤매너 (Tone & Manner)

| 키워드 | 설명 |
|---|---|
| **Bright** | 밝고 환한. 어두운 검은 터미널 이미지를 피하고, 흰/밝은 배경 위에 따뜻한 색을 얹는다. |
| **Friendly** | 친근한. "어려운 도구"라는 인식을 지우고, "재미있는 장난감"처럼 다가간다. |
| **Tutorial** | 한 단계 한 단계. 스크롤이 곧 학습 진행 바. |
| **Dribbble/Notion-grade** | 시각적 완성도. 정돈된 여백, 섬세한 일러스트, 부드러운 모션. |
| **Warm, not cold** | 차가운 기술 느낌이 아니라 따뜻한 종이/나무/포스트잇 질감의 온기. |

**안티 테마(피할 것)**:
- 다크 모드 기본 / 네온 / 사이버펑크 / "해커" 미학.
- 폭탄 같은 정보 밀집. 여백이 부족하면 비개발자는 바로 이탈한다.

## 2. 컬러 시스템 (Korean rationale)

> 전체 톤은 "햇살 드는 방" — 따뜻하고 편안한.

| 역할 | 색상(방향) | 한국어 이름 | 비고 |
|---|---|---|---|
| Background-base | `#FFFBF5` 계열 | "따뜻한 크림색" | 순백(#FFF)은 차가워 보여 미색으로. |
| Background-surface | `#FFFFFF` | "카드 하양" | 카드/박스 위에만 순백. |
| Primary | `#FF8A5B` 계열 | "따뜻한 코랄" | tmux의 기본 상태표시줄 초록을 오마주하되, 더 부드럽게. |
| Accent (키/명령어) | `#3D5AFE` 연한 톤 | "편안한 인디고" | 단축키/코드는 차갑지 않은 진파랑으로. monospace와 짝꿍. |
| Secondary | `#26A69A` 계열 | "민트 팁" | 팁 박스/성공 상태. |
| Warning | `#EF5350` 부드러운 톤 | "코랄 레드(주의)" | 주의 박스. 자극적 빨강 금지. |
| Text-primary | `#2D2A26` | "따뜻한 먹색" | 검정(#000) 대신 먹색. |
| Text-secondary | `#7A726B` | "회갈색" | 본문 보조 텍스트. |

**side-note 박스 색 매핑**(content-plan.md의 5종 박스와 대응):
- 비유 박스 → 따뜻한 노랑(`#FFF7E6` 배경)
- 역사 박스 → 빈티지 베이지(`#F5EFE0`)
- 팁 박스 → 민트(`#E6F7F4`)
- 주의 박스 → 부드러운 코랄(`#FDECEA`)
- "알아두면 좋은" 박스 → 연한 라벤더(`#F1ECFB`)

## 3. 타이포그래피 (Typography)

| 역할 | 폰트(방향) | 이유 |
|---|---|---|
| 한국어 본문/헤드라인 | Pretendard / Spoqa Han Sans Neo 계열 | 가독성 좋은 현대적 한국어 sans. 굵기 변화로 위계. |
| 영문 본문 | Inter / Nunito Sans | 둥글고 부드러운 — 친근함. |
| 코드/단축키/명령어 | JetBrains Mono / Fira Code | monospace. `C-b d` 같은 키 표시 전용. |
| 디스플레이(큰 헤드라인) | 둥근 geometric sans (Pretendard Black / Nunito ExtraBold) | 튜토리얼의 부드러움. |

**위계(방향)**:
- Hero 헤드라인: 56~72px / 800 weight
- 섹션 헤드라인: 36~44px / 700 weight
- 서브헤드: 22~26px / 600
- 본문: 17~18px / 400 / line-height 1.7
- 단축키/코드: 15~16px monospace
- side-note: 15px / 400

**단축키 표시 컴포넌트("key cap")**:
- `C-b` 따위를 물리 키보드 키 모양(kbd 스타일)으로 표현.
- 약간의 그림자 + 둥근 모서리로 "누를 수 있는" 느낌.

## 4. 레이아웃 시스템 (Single-Page Scroll)

```
┌──────────────────────────────────────────────┐
│ [Sticky Nav]  로고 · 섹션바로가기 · 진행률   │  ← 상단 고정
├──────────────────────────────────────────────┤
│ [Reading Progress Bar]  ─────────█── 42%      │  ← 맨 위 2px
├──────────────────────────────────────────────┤
│                                              │
│  [0] HERO                                    │
│   ┌─────────────────────────────────────┐    │
│   │  큰 일러스트 + 헤드라인 + 스크롤유도 │    │
│   └─────────────────────────────────────┘    │
│                                              │
│  [1] CONCEPT                                 │
│   본문 │ [side-note 박스]                     │  ← 2단: 본문 + 곁가지
│                                              │
│  ...                                         │
│                                              │
│  [4] MENTAL MODEL                            │
│   ┌─────────────────────────────────────┐    │
│   │ [Animated Hierarchy Diagram]         │    │  ← 전폭 인터랙티브
│   └─────────────────────────────────────┘    │
│                                              │
│  [5] CORE USAGE                              │
│   [Interactive Fake-Terminal Demo]           │  ← 버튼 → 분할 애니
│                                              │
│  [6] KEYBINDINGS                             │
│   [Sticky Cheat-Sheet Card (또는 모달)]       │
│                                              │
│  [10] APPLICATIONS                           │
│   [Before/After 탭: 일반 터미널 vs tmux]      │
│                                              │
│  [13] FOOTER                                 │
└──────────────────────────────────────────────┘
```

**레이아웃 원칙**:
- **콘텐츠 최대 폭**: ~720~840px (가독성). 인터랙티브 데모/다이어그램은 전폭.
- **데스크톱**: 본문 + 곁가지 side-note 2단(또는 곁가지는 인라인 카드).
- **모바일**: 1단. side-note는 본문 흐름 안으로 접기/펼치기.
- **섹션 간 여백**: 넉넉하게(120~160px). "쉬어가는 공기"가 튜토리얼 느낌을 만든다.

## 5. 핫 에셋 인벤토리 ("Hot Assets")

> 각 에셋은 **비개발자 학습 가치**로 정당화되어야 한다. 과잉 엔지니어링 금지.

### A. Scroll-triggered Reveal Animations
- **목적**: 스크롤하면 개념이 "나타나는" 모션. 읽는 속도에 맞춰 정보가 펼쳐짐.
- **비개발자 학습 가치**: 한꺼번에 쏟아지면 압도되는 비개발자에게 "한 입 크기"로 정보를
  나눠줌. 인지 부하를 낮춤.
- **기술(방향)**: 순수 JS `IntersectionObserver` + CSS `transform/opacity`. 라이브러리 없음.
- **적용 지점**: 모든 섹션 헤드라인, 다이어그램 요소, 단계별 목록.

### B. Interactive Fake-Terminal Demo
- **목적**: 가짜 터미널 박스. 버튼 클릭 → prefix 키 누름 → pane 분할 애니메이션이 재생됨.
- **비개발자 학습 가치**: 실제 터미널을 켤 필요 없이 "tmux가 어떻게 반응하는지"를 눈으로 체험.
  "C-b %를 누르면 화면이 좌우로 갈라진다"를 행동으로 이해시킴.
- **기술(방향)**: 순수 CSS로 터미널 외관 + JS로 키 입력 시퀀스 + CSS transition으로 분할.
  실제 shell 없음. 모든 상태는 미리 정의된 스크립트.
- **데모 시나리오**:
  1. "세션 시작" 버튼 → 가짜 터미널에 `$ tmux` 타이핑 → 상태표시줄 등장.
  2. "분할(좌우)" 버튼 → `C-b %` 시퀀스 재생 → 화면 좌우 분할 애니메이션.
  3. "분할(상하)" / "pane 이동" 버튼 → 각각 대응 모션.
  4. "detach" 버튼 → 화면이 "접히고" → "밖으로" 메시지 → "attach" 버튼으로 복귀.

### C. Animated Hierarchy Diagram (server/session/window/pane)
- **목적**: 4층 구조를 건물 비유로 시각화. 스크롤/클릭에 따라 층이 확장됨.
- **비개발자 학습 가치**: tmux의 가장 큰 진입장벽인 "계층 모델"을 한 장의 그림으로 해소.
  비유(건물/방/책상/쪽지)와 실제 용어가 같은 그림에 공존 → 브릿지 효과.
- **기술(방향)**: 인라인 SVG 또는 CSS positioned divs. 스크롤 진행도에 따라 층이 하나씩 펼쳐짐.
- **애니메이션 흐름**: 건물(서버) → 방 세션들(session) 나열 → 한 방 클릭/스크롤 → 책상(window)들 →
  한 책상 클릭 → 쪽지(pane) 분할 모습.

### D. Warm Friendly Illustration Set
- **목적**: 핵심 비유(방/책상/창문/포스트잇)를 따뜻한 일러스트로 표현.
- **비개발자 학습 가치**: 글만 읽으면 추상적인 개념이 일러스트로 "구체적 경험"이 됨.
  비유가 시각화되면 기억에 오래 남음.
- **필수 일러스트 목록**:
  - 여러 창문이 달린 따뜻한 방 (Concept)
  - 건물 단면도 (Mental Model)
  - 책상 위 포스트잇 여러 장 (pane 분할)
  - 현관문을 닫고 나가는 사람 + 방 안엔 불이 켜진 채 (detach)
  - 다시 문을 열고 들어오는 사람 (attach)
  - 두 사람이 같은 방을 들여다보는 모습 (pair programming)
- **스타일**: 플랫 + 약간의 텍스처, 따뜻한 팔레트, 둥근 형태. unDraw / Storyset급 톤.

### E. Sticky Keybinding Cheat-Sheet + Reading-Progress Bar
- **목적**: (1) 페이지 최상단 2px 진행률 바. (2) 화면 한쪽에 떠 있는(또는 토글되는) 단축키 카드.
- **비개발자 학습 가치**: "읽고 있는 위치"를 항상 보여주어 긴 페이지에서 길을 잃지 않게 함.
  치트시트를 언제든 펼쳐보면 "지금 읽은 개념의 단축키가 뭐였지?"를 즉시 해결.
- **기술(방향)**: 진행률 바 = scroll event + width %. 치트시트 = CSS sticky / position fixed +
  토글 버튼.

### F. Before/After or Tabbed Comparison (일반 터미널 vs tmux)
- **목적**: 같은 작업을 (a) 일반 터미널 (b) tmux 로 했을 때의 차이를 나란히/탭으로 비교.
- **비개발자 학습 가치**: "왜 tmux가 나은지"를 말로 설명하는 것보다 나란히 보는 것이 압도적으로
  빠름. 특히 Applications 섹션의 SSH 지속성 시나리오에서 효과 만점.
- **기술(방향)**: CSS grid 2단 비교 + 슬라이더 드래그(before/after) 또는 탭 전환.

### G. Gentle Onboarding Micro-interactions
- **목적**: 작은 호버/클릭 피드백, "키 캡" 눌림 효과, 카드 살짝 떠오름 등.
- **비개발자 학습 가치**: 인터랙션이 부드러우면 "이 사이트는 나를 배려한다"는 느낌 → 학습 의지
  상승. 어설픈 micro-animation은 오히려 산만하므로 최소한으로.
- **기술(방향)**: CSS transition / hover / active 상태. JS 최소화.

### H. (보너스) " detach 공포" 안심 인터랙션
- **목적**: FAQ Q4에서 "detach해 버렸어요!" 시나리오를 인터랙티브으로 재현 → "다시 attach" 버튼
  누르면 화면이 복귀하는 모션.
- **비개발자 학용 가치**: 가장 흔한 초보자 공포를 직접 체험하고 해소 → "별거 아니었네"라는 깨달음.
- **기술(방향)**: fake-terminal demo의 변형. 상태 머신 2단계(detached → attached).

## 6. 접근성 & 모바일 (Accessibility & Mobile)

| 항목 | 방향 |
|---|---|
| 키보드 탐색 | 모든 인터랙티브 요소 tab 가능. 데모 버튼 Enter/Space 동작. |
| 스크린 리더 | 일러스트에 `alt`. side-note는 `aside`/`role="complementary"`. 진행률 바 `aria-valuenow`. |
| 색 대비 | WCAG AA 준수(본문 4.5:1 이상). 따뜻한 톤이라도 대비는 희생하지 않음. |
| 모션 민감도 | `prefers-reduced-motion: reduce`에서 reveal/diagram 모션 축소 또는 정지. |
| 모바일 | 1단 레이아웃. side-note 인라인 접기. 치트시트는 하단 시트로. 데모 터치 동작 지원. |
| 폰트 크기 | 본문 최소 16px. 모바일에서도 가독성 유지. |
| 한국어 줄바꿈 | `word-break: keep-all`로 단어 중간 끊김 방지. |
