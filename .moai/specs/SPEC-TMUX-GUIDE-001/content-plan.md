# 콘텐츠 플랜 — 비개발자를 위한 tmux 단일 페이지 학습 가이드

> 본 파일은 **기획 산출물** 이다. 코드 없음. 본문의 한국어 카피는 "방향(direction)"이며,
> 실제 라이팅은 이후 구현 Phase에서 다듬어진다. 모든 tmux 팩트는 공식 위키 기준으로
> 검증되었다(prefix 기본 `C-b`, 분할 `C-b %` / `C-b "`, detach `C-b d` 등).

---

## 1. 정보 구조 (Information Architecture)

단일 스크롤 페이지. 다음 순서대로 전개된다(Arc: Concept → Fundamentals → Practice → Applications).

```
[0] HERO           — 한 줄 약속 + 스크롤 유도
[1] CONCEPT        — tmux가 뭔가? (비유 먼저, 단어 나중)
[2] WHY            — 왜 쓰나? (5가지 상황)
[3] INSTALL        — 설치 (mac/Linux/WSL) + prefix 키 개념
[4] MENTAL MODEL   — server/session/window/pane (계층 비유)
[5] CORE USAGE     — 세션/분할/이동/크기조절 (cheat-sheet 카드)
[6] KEYBINDINGS    — 단축키 치트시트 (스티키 카드)
[7] COPY MODE      — 마우스 없이 복사/스크롤
[8] CONFIG         — .tmux.conf / 상태표시줄 꾸미기
[9] PLUGINS & TPM  — 테마, 세션 저장/복원(resurrect/continuum)
[10] APPLICATIONS  — 실생활 워크플로 5종
[11] TROUBLESHOOT  — FAQ / "세션 잃어버림" 공포 해소
[12] DEEP DIVE     — (곁가지 정리) 역사/비교/철학
[13] FOOTER        — 다음 단계/참고링크
```

### 왜 이 순서인가 (Narrative Arc Rationale)

1. **Concept → Why**: 독자가 "뭔지" 모르면 "왜 쓰는지" 와닿지 않는다. 먼저 비유로 그림을 그리고,
   그 그림 위에서 "왜 유용한지"를 증명한다.
2. **Install → Mental Model**: 손에 넣기(설치) 직후, "이제 뭘 보고 있는 거지?"를 해소하기 위해
   계층 모델(server/session/window/pane)을 비유로 제시한다. 그래야 이후 명령이 "무엇을 조작하는지" 안다.
3. **Core Usage → Keybindings → Copy Mode**: 조작 → 단축키 집중 → 마우스 없는 복사 순으로
   실습 난이도가 자연스럽게 올라간다.
4. **Config → Plugins**: 기본 사용법이 익숙해진 뒤에야 "꾸미기"와 "확장"이 의미를 갖는다.
5. **Applications 마지막 배치**: 모든 개념이 쌓인 상태에서 실생활 시나리오가 "아하"로 연결된다.
   특히 SSH 지속성은 Concept/Why의 약속을 Applications에서 "보상"으로 돌려주는 클라이맥스.
6. **Troubleshoot → Deep Dive**: 공포(세션 잃어버림)를 먼저 달래고, 호기심 많은 독자에게는
   곁가지(역사/비교/철학)를 마지막 보너스로 제공한다.

## 3. 섹션별 카피 방향 (Sample Korean Copy)

> 아래 헤드라인/서브헤드/본문 1~2문장은 톤을 보여주는 방향이다. 모든 기술어 앞에 비유가 선행된다.

### [0] HERO
- **헤드라인(방향)**: "터미널 하나로, 방 하나를 통째로 쓰는 법"
- **서브헤드(방향)**: "컴퓨터를 끄지 않아도, 인터넷이 끊겨도, 내 작업은 그 자리에 살아 있어요."
- **본문(방향)**: "개발자들이 매일 쓰는 도구 tmux를, 코딩을 모르는 분도 한 번에 이해하도록 풀어냈어요. 스크롤 한 번이면 충분합니다."
- **에셋**: 따뜻한 일러스트(여러 창이 떠 있는 책상) + 아래로 스크롤 유도 인디케이터.

### [1] CONCEPT — tmux가 뭔가?
- **헤드라인(방향)**: "창문이 많은 방, 그리고 창문을 더 만들 수 있는 책상"
- **서브헤드(방향)**: "tmux는 한 화면 안에서 여러 화면을 동시에 띄워두는 도구예요."
- **본문(방향)**: "먼저 비유로 시작할게요. 컴퓨터 화면 하나를 '방'이라 상상해 보세요. 방에는 창문이 여러 개 달릴 수 있고, 창문 하나하나에는 서로 다른 풍경이 보여요. tmux는 이 방 안에서 창문을 마음껏 늘리거나, 나누거나, 다른 쪽으로 돌려놓을 수 있게 해 주는 '창문 관리 도구'예요."
- **연결 용어 도입**: 방 → 터미널 창(window), 창문 안 조각 → pane, 창문을 달았다 뗐다 하는 행위 → detach/attach.
- **용어 브릿지(방향)**: "이 방을 관리하는 큰 시스템이 바로 **터미널 멀티플렉서(terminal multiplexer)**. tmux는 그중에서도 가장 사랑받는 한 종류예요."

**연결 side-note**:
- **역사 박스(옛날 이야기)**: "옛날에는 `screen`이라는 도구가 먼저 있었어요. 1987년에 태어난 이 도구의 아이디어를 2007년에 Nicholas Marriott가 더 modern하게 다듬어 태어난 게 바로 tmux(tee-mux)입니다."
- **비유 박스(이렇게 생각해보세요)**: "terminal / shell / multiplexer가 헷갈린다면 — terminal은 '창문 틀', shell은 '창문 너머로 말을 거는 비서', multiplexer는 '창문 틀을 통째로 관리하는 건축가'라고 생각하면 쉬워요."
- **"알아두면 좋은" 박스**: "터미널은 글자만 보이는 검은 화면이에요. 마우스를 잘 안 쓰고 키보드로 모든 걸 합니다. 처음엔 어색하지만, 익숙해지면 마우스보다 빠르고 편해져요."

### [2] WHY — 왜 쓰나?
- **헤드라인(방향)**: "왜 다들 tmux를 좋아할까요?"
- **5가지 상황(각각 카드형)**:
  1. **지속되는 작업(persistent sessions)** — "방에 불을 켜둔 채로 밖에 나갈 수 있어요." 인터넷이 끊겨도 작업은 방 안에 살아 있음.
  2. **화면 분할(split screen)** — "한 창에서 코드도 보고, 설명서도 보고, 채팅도 볼 수 있어요."
  3. **원격 작업 생존(remote survival)** — "노트북을 덮어도 서버에선 작업이 계속돼요."
  4. **페어 프로그래밍(pair programming)** — "같은 화면을 두 사람이 같이 볼 수 있어요."
  5. **멀티태스킹(multi-tasking)** — "여러 창을 번갈 보며 한 번에 여러 일을 해요."

**연결 side-note**:
- **비유 박스**: "옛날 컴퓨터는 화면이 하나뿐이었어요. 한 번에 하나만 할 수 있었죠. tmux는 그 하나의 화면을 여러 조각으로 쪼개 쓰는 아이디어에서 출발했어요."
- **"알아두면 좋은" 박스**: "개발자들이 tmux를 사랑하는 진짜 이유는 '맥락(context)을 잃지 않는 것'이에요. 무엇을 하고 있었는지, 어디까지 했는지 화면이 그대로 남아 있으니까요."

### [3] INSTALL — 설치 + prefix 키 개념
- **헤드라인(방향)**: "설치해 봐요 (3분 컷)"
- **플랫폼별(카드형, 각각 한 줄 명령어 + "이걸 치면 뭐가 되나요" 설명)**:
  - **mac**: `brew install tmux` — "Homebrew라는 앱스토어 같은 도구로 한 줄 설치."
  - **Linux (Ubuntu/Debian)**: `sudo apt install tmux` — "리눅스의 앱스토어 명령어."
  - **Windows (WSL)**: WSL 설치 후 리눅스와 동일 — "윈도우 안에 작은 리눅스 방을 하나 더 달아서 그 안에서 tmux를 써요."

**prefix 키 개념 도입 (핵심 — 부드럽게)**:
- **비유 박스(이렇게 생각해보세요)**: "tmux한테 말을 걸려면 먼저 '종을 울려야' 해요. 그 종이 바로 **prefix 키**. 기본 종은 `Ctrl`과 `b`를 동시에 누르는 `C-b`예요. 종을 울린 뒤에야 '창을 만들어줘(c)'처럼 명령을 내릴 수 있어요."
- **주의 박스(잠깐, 주의!)**: "`C-b c`는 'Ctrl과 b를 누르고, 손을 뗀 뒤, c를 누른다'는 뜻이에요. `Ctrl`을 누른 채로 c까지 누르는 게 아니에요! 처음엔 다들 헷갈려요."

### [4] MENTAL MODEL — server/session/window/pane
- **헤드라인(방향)**: "tmux의 4층 구조, 비유로 한 번에 정리"
- **본문(방향)**: "tmux 안에는 4개의 개념이 층층이 쌓여 있어요. 헷갈리지 않게, '건물'에 빗대어 볼게요."

| tmux 용어 | 일상 비유 | 한 줄 설명 |
|---|---|---|
| **server** | 건물의 관리사무소 | 보이지 않는 곳에서 모든 걸 총괄. 전원이 꺼지지 않는 한 계속 돌아감. |
| **session** | 건물 안의 방 하나 | 독립된 작업 공간. 방 여러 개를 만들 수 있고, 방마다 다른 일을 함. |
| **window** | 방 안의 책상 | 한 방에 책상(탭)이 여러 장. 번호(0,1,2...)로 구분. |
| **pane** | 책상 위의 쪽지들 | 한 책상을 여러 조각으로 쪼갠 각 조각. 각 조각마다 다른 프로그램이 돌아감. |

- **에셋**: 이 표를 **애니메이션 계층 다이어그램**(건물 → 방 → 책상 → 쪽지가 확장되는 모션)으로 시각화.
- **연결 용어 강조**: "active pane" = 지금 손을 대고 있는 쪽지 / "current window" = 지금 앉아 있는 책상.

**연결 side-note**:
- **비유 박스**: "session을 방, window를 책상이라 하면, detach는 '방에서 나와 현관문을 닫는 것', attach는 '다시 현관문을 열고 들어가는 것'이에요. 방 안의 물건(작업)은 그대로 남아 있어요."
- **"알아두면 좋은" 박스**: "방(session)에 불을 끄지 않는 한, 그 안에서 돌아가는 프로그램은 계속 돌아가요. 이게 tmux의 가장 강력한 점이에요."

### [5] CORE USAGE — 핵심 사용법
- **헤드라인(방향)**: "이것만 알면 tmux 80%는 써요"
- **카드형 목록(각 카드: 비유 1문장 → 단축키 → 결과 설명)**:

| 하고 싶은 일 | 비유 | 단축키 | 결과 |
|---|---|---|---|
| 세션 시작 | 방 만들기 | (터미널에) `tmux` 또는 `tmux new -s 이름` | 방 하나가 열리고 안으로 들어감 |
| 세션에서 나오기(detach) | 방문을 닫고 나오기 | `C-b d` | 방은 그대로, 나만 밖으로 |
| 다시 들어가기(attach) | 다시 방문 열기 | (터미널에) `tmux attach` 또는 `tmux a` | 마지막 방으로 복귀 |
| 방 목록 보기 | 건물 층 안내도 | (터미널에) `tmux ls` | 현재 열려 있는 방들 목록 |
| 창(책상) 만들기 | 책상 한 장 더 | `C-b c` | 새 책상(번호+1)이 열림 |
| 창 이동 | 책상 옮겨 앉기 | `C-b 0`~`C-b 9`, `C-b n`(다음), `C-b p`(이전) | 해당 책상으로 이동 |
| 창 분할(좌우) | 책상을 좌우로 자르기 | `C-b %` | 현재 조각이 좌우 두 개로 |
| 창 분할(상하) | 책상을 위아래로 자르기 | `C-b "` | 현재 조각이 위아래 두 개로 |
| 조각(pane) 이동 | 다른 쪽지로 손 옮기기 | `C-b ←/→/↑/↓` | 화살표 방향의 쪽지로 |
| 조각 크기 조절 | 쪽지를 더 넓히기/줄이기 | `C-b C-←/→/↑/↓` (작게), `C-b M-←/→/↑/↓` (크게) | 경계선이 밀림 |
| 쪽지 번호 보기 | "몇 번 쪽지?" 확인 | `C-b q` | 각 쪽지 위에 번호가 잠깐 뜸 |
| 전체 확대/복귀(zoom) | 한 쪽지를 책상 전체로 | `C-b z` | 다시 누르면 원래대로 |
| 창/쪽지 닫기 | 책상/쪽지 치우기 | `C-b &`(창), `C-b x`(쪽지) | 닫기 전 확인 프롬프트 |

**연결 side-note**:
- **팁 박스(꼭 알아두면 좋아요)**: "번호가 헷갈릴 땐 `C-b q`! 각 쪽지 위에 큰 숫자가 잠깐 뜨면서, 그 숫자를 누르면 바로 그 쪽지로 이동해요."
- **주의 박스(잠깐, 주의!)**: "`C-b d`로 나가도 방(session)은 절대 사라지지 않아요. '닫은 게 아니라 잠시 나온 것'이에요. 다시 `tmux a` 치면 그대로 있어요."
- **비유 박스**: "화면 분할은 책상을 자르는 것과 같아요. 같은 책상 위에 두 조각이 나란히 놓이는 거죠. 한 조각에서 글을 쓰면서 다른 조각의 자료를 볼 수 있어요."

### [6] KEYBINDINGS — 치트시트
- **헤드라인(방향)**: "단축키 카드, 한눈에"
- **형태**: 스티키(화면에 떠 있는) 치트시트 카드. 읽기 진행률과 함께 표시.
- **내용**: 위 [5] 표를 치트시트 디자인으로 재구성. prefix `C-b`는 별도 강조(큰 글씨).
- **연결 side-note**:
  - **팁 박스**: "prefix 키를 `C-a`로 바꾸는 사람이 많아요. 손이 더 편하거든요. 바꾸는 법은 뒤에 Config에서!"
  - **"알아두면 좋은" 박스**: "`C-b ?`를 누르면 모든 단축키가 리스트로 떠요. 외울 필요 없이 그때그때 찾아보면 돼요."

### [7] COPY MODE — 마우스 없이 복사/스크롤
- **헤드라인(방향)**: "마우스 없이 글자를 복사하고, 위로 스크롤해요"
- **본문(방향)**: "터미널에서 위로 스크롤하려고 마우스 휠을 굴려도 안 움직여서 당황한 적 있죠? tmux에서는 '복사 모드'라는 특별한 모드로 들어가야 해요."
- **절차(단계형)**:
  1. `C-b [` — 복사 모드 진입. (비유: "돋보기를 꺼내 드는 단계")
  2. 화살표/`Page Up`/`Page Down`으로 이동.
  3. (emacs 모드 기준) `C-Space`로 선택 시작 → 커서 이동 → `C-w`로 복사 후 모드 종료. (vi 모드: `Space` 시작, `Enter` 복사)
  4. `C-b ]` — 복사한 것 붙여넣기.
- **에셋**: 인터랙티브 가짜 터미널 데모(버튼 누르면 복사 모드 진입 애니메이션).

**연결 side-note**:
- **비유 박스**: "복사 모드는 '잠시 화면을 정지시키고 돋보기로 살펴보는 시간'이에요. 보통 모드에서는 새 글자가 계속 올라오지만, 복사 모드에선 화면이 멈춰서 자유롭게 훑어볼 수 있어요."
- **주의 박스**: "복사 모드를 끝내려면 `q`! 빠져나오는 법을 잊으면 당황하기 쉬워요."
- **팁 박스**: "마우스를 쓰고 싶다면 `.tmux.conf`에 `set -g mouse on`을 한 줄 추가하면 휠 스크롤과 드래그 복사가 가능해져요."
- **"알아두면 좋은" 박스**: "복사한 글자는 tmux만의 '버퍼'에 보관돼요. 최대 50개까지 저장되고, `C-b =`로 과거 버퍼 목록을 볼 수 있어요."

### [8] CONFIG — .tmux.conf / 상태표시줄
- **헤드라인(방향)**: "나만의 tmux로 꾸미기"
- **본문(방향)**: "tmux는 집처럼 꾸밀 수 있어요. 집의 설계도 역할을 하는 파일이 바로 홈 폴더의 `.tmux.conf`예요."
- **기본 예시 방향(비개발자에게 읽히도록 주석 달아 설명)**:
  - prefix 키 바꾸기 (`C-a`로)
  - 마우스 켜기 (`set -g mouse on`)
  - 상태표시줄 색/내용 바꾸기 (`status-style`, `status-left`/`status-right`)
  - 창 번호를 1부터 시작 (`set -g base-index 1`)
- **상태표시줄(status line) 설명**: "화면 맨 아래 초록 줄이 보이죠? 왼쪽엔 방 이름, 가운데엔 책상 목록, 오른쪽엔 시간이 떠요. 이것도 전부 내 마음대로 바꿀 수 있어요."
- **에셋**: Before/After 비교(기본 상태표시줄 vs 커스텀 상태표시줄).

**연결 side-note**:
- **팁 박스**: "설정 파일을 고치고 나서 바로 적용하려면 tmux 안에서 `C-b :`로 명령 프롬프트를 열고 `source ~/.tmux.conf`를 입력하면 돼요. tmux를 다시 켤 필요 없어요."
- **"알아두면 좋은" 박스**: "tmux 설정은 4가지 종류(server/session/window/pane)가 있어요. 보통은 전체에 적용되는 global 설정만 만져도 충분해요."

### [9] PLUGINS & TPM — 테마, 세션 저장/복원
- **헤드라인(방향)**: "tmux에 '앱'을 더해요: 플러그인과 tpm"
- **본문(방향)**: "스마트폰에 앱을 설치하듯, tmux에도 플러그인을 추가할 수 있어요. 이걸 관리해 주는 앱스토어 역할이 **tpm(tmux plugin manager)** 이에요."
- **대표 플러그인 3종(비개발자용 설명)**:
  - **테마 플러그인**(예: tmux-themepack, catppuccin/tmux 등) — "상태표시줄과 색을 예쁘게 바꿔주는 스킨."
  - **tmux-resurrect** — "방 안의 책상 배치까지 기억했다가, 컴퓨터를 껐다 켜도 그대로 복원해 주는 '저장' 기능."
  - **tmux-continuum** — "resurrect가 저장한 걸 주기적으로 자동 저장해 주는 비서. 컴퓨터가 꺼져도 마지막 상태가 살아 있어요."
- **설치 흐름(개념도)**: tpm 설치 → `.tmux.conf`에 원하는 플러그인 나열 → `prefix + I`(설치) → 완료.

**연결 side-note**:
- **"알아두면 좋은" 박스**: "resurrect와 continuum을 같이 쓰면, '내 노트북이 꺼져도 서버 작업은 살아있다'는 약속이 진짜로 현실이 돼요. tmux의 궁극의 무기예요."
- **주의 박스**: "플러그인이 너무 많으면 tmux가 느려질 수 있어요. 정말 필요한 것부터 하나씩!"

### [10] APPLICATIONS — 실생활 워크플로 5종
- **헤드라인(방향)**: "실제로 어떻게 쓰이나요? 다섯 가지 이야기"
- **시나리오 카드(각각: 상황 → tmux가 어떻게 도와주는지 → "아하" 포인트)**:
  1. **SSH 원격 세션 지속** — "회사 서버에 접속해 작업하다 와이파이가 끊겨도, 다시 접속해서 `tmux a` 한 줄이면 그 자리로 복귀."
  2. **페어 프로그래밍** — "두 사람이 같은 tmux session에 붙어서 한 화면을 같이 보며 코딩."
  3. **로그 모니터링(여러 pane)** — "한 화면에 서버 로그/에러/현재 작업을 동시에 띄워두고 한눈에 감시."
  4. **멀티 모니터 대체** — "모니터가 하나여도 tmux 분할로 여러 창을 동시에."
  5. **'내 노트북이 꺼져도 서버 작업은 살아있다'** — "클라이맥스 시나리오. 노트북을 덮고 퇴근해도, 서버의 tmux session은 계속 돌아가고 있어요."

**연결 side-note**:
- **비유 박스**: "tmux 없이 SSH를 쓰면, 인터넷이 끊기는 순간 서버에서 하던 일도 같이 사라져요. 마치 전화를 끊으면 통화 내용이 증발하는 것처럼요. tmux는 '통화 내용이 녹음되어 남는 전화'를 만들어 주는 도구예요."
- **"알아두면 좋은" 박스(보안)**: "공유 session은 강력하지만 주의가 필요해요. 같은 session에 붙은 사람은 내가 치는 모든 걸 볼 수 있거든요. 믿을 수 있는 사람과만!"

### [11] TROUBLESHOOTING — FAQ
- **헤드라인(방향)**: "당황하지 마세요, 다 해결책이 있어요"
- **FAQ 카드**(각각: 증상 → 원인 → 해결):
  - **Q: 스크롤이 안 돼요!** → A: 기본 상태. `C-b [`로 복사 모드 진입 후 스크롤. 또는 `.tmux.conf`에 `set -g mouse on`.
  - **Q: 마우스로 클릭/드래그가 안 돼요.** → A: mouse 옵션이 꺼져 있음. `set -g mouse on`.
  - **Q: 색이 이상하게 나와요.** → A: `default-terminal` 설정(`set -s default-terminal 'tmux-256color'`).
  - **Q: detach해 버렸어요, 작업 날아갔나요?!** → A: 아님! `tmux a`로 그대로 복귀. (공포 해소 박스)
  - **Q: 세션을 잃어버렸어요.** → A: `tmux ls`로 확인, `tmux a -t 이름`으로 재진입.
  - **Q: tmux 자체를 완전히 끄고 싶어요.** → A: `tmux kill-server`. 단, 모든 세션이 날아감(주의 박스).

**연결 side-note**:
- **주의 박스(잠깐, 주의!)**: "`kill-server`는 건물(서버) 자체를 허무는 거예요. 모든 방과 작업이 즉시 사라져요. 정말 다 필요 없을 때만!"
- **팁 박스**: "tmux 안에서 뭔가 이상하면 `C-b ?`(단축키 목록), `C-b :`(명령 프롬프트)로 항상 도움을 받을 수 있어요."

### [12] DEEP DIVE — 곁가지 정리 (보너스)
- **헤드라인(방향)**: "더 깊이 알고 싶은 분을 위해"
- **하위 주제(탭/아코디언형)**:
  - **터미널 멀티플렉서의 역사**: `screen`(1987) → `tmux`(2007, OpenBSD 프로젝트에서 파생).
  - **왜 개발자들이 tmux에 집착할까**: "맥락 보존(context preservation)"이라는 철학.
  - **tmux vs screen vs zellij vs iTerm2 탭 비교**: 표로 정리(지속성/분할/원격/플랫폼).
  - **공유 세션 보안 노트**: pair programming 시 민감 정보 주의.
  - **생산성 철학**: "화면을 분할하기 전에, 내가 무엇에 집중하고 있는지" — 도구 이전의 마인드셋.

**연결 side-note**:
- **역사 박스(옛날 이야기)**: "tmux는 2007년 OpenBSD 운영체제 개발 과정에서 태어났어요. '화면 관리 도구가 더 필요하다'는 요구에서 시작됐죠."
- **"알아두면 좋은" 박스**: "요즘은 zellij라는 '차세대 tmux'를 만들려는 시도도 있어요. 하지만 tmux는 20년 가까운 검증과 엄청난 커뮤니티 자료를 가진, 여전히 사실상 표준이에요."
