# 사례 갤러리 카탈로그 (Cases Gallery) — 비개발자를 위한 Git/GitHub 가이드 시그니처

> 본 사이트의 **시그니처 기능**. "아\~ 이렇게도 쓸 수 있구나" 순간을 설계. 두 티어: Tier 1 (비개발자 실생활) + Tier 2 (전문가/오픈소스). 각 사례는 (a) 비개발자용 설명 + (b) 어떤 git/github 기능을 보여주는지 + (c) 어느 튜토리얼 페이지로 연결되는지를 포함.

---

## 1. 사례 갤러리 철학

사용자가 명시적으로 강조: "아\~ 이렇게도 쓸 수 있구나" — 사례는 시그니처.

- **Tier 1 (비개발자 실생활)**: "Git/GitHub가 코딩이 아닌 일상에서 어떻게 쓰이는지"를 보여줌. 비개발자 독자에게 "이건 내 세계 이야기"라는 동기부여.
- **Tier 2 (전문가/오픈소스)**: "Git/GitHub가 실제 산업·학계·공공에서 어떻게 쓰이는지". "이 도구가 세상을 어떻게 바꿨나"라는 거시적 시각.

각 카드는 튜토리얼 페이지로 양방향 연결 (사례 콜아웃 박스 ↔ 갤러리 카드).

---

## 2. Tier 1 — 비개발자 실생활 사례

> "코딩을 모르는 분도 매일 Git/GitHub를 쓸 수 있어요."

### T1-01. 소설가·작가 — 원고 버전 관리

- **한 줄 설명(방향)**: "한 챕터를 고칠 때마다 '최종', '진짜최종' 파일 대신, 커밋 한 번이면 과거 어떤 버전으로든 돌아갈 수 있어요. 엇갈리는 두 결말? 브랜치로 평행 우주처럼 따로 써요."
- **보여주는 기능**: commit(스냅샷), branch(대안 시나리오), checkout(과거 버전 돌아가기), log(수정 이력).
- **연결 튜토리얼**: T01(concept), T03(commit), T06(branch), T11(undo).
- **일러스트**: 원고 더미 위에 사진첩이 펼쳐진 모습, 두 갈래 결말이 나뭇가지로 갈라짐.

### T1-02. 디자이너 — 에셋·SVG·Figma 내보내기 히스토리

- **한 줄 설명(방향)**: "로고 SVG를 100번 고쳤는데, 47번째 버전이 필요해요? blame/checkout으로 바로. 디자인 시스템도 버전별로 관리해 두 번 다시 '이 색 뭐였지?' 안 해도 돼요."
- **보여주는 기능**: log, blame(이 줄 누가 언제), checkout(특정 버전), tag(디자인 시스템 v1/v2).
- **연결 튜토리얼**: T05(history), T10(tag), T13(.gitignore·LFS).
- **일러스트**: 색 팔레트와 SVG 파일들이 사진첩 페이지에 정리된 모습.

### T1-03. 연구자·학생 — 논문 + 데이터셋 + 재현 가능 연구

- **한 줄 설명(방향)**: "논문 초고 + 실험 데이터 + 분석 스크립트를 한 저장소에. '이 결론 어떤 데이터에서 나왔어?' → 그 커밋으로 점프. 연구 재현성이 한층 올라가요."
- **보여주는 기능**: commit, LFS(큰 데이터셋), tag(논문 제출 버전), blame(결론 출처 추적).
- **연결 튜토리얼**: T03, T05, T10, T13(LFS).
- **일러스트**: 실험 노트 + 그래프 + 데이터 파일이 한 사진첩에.

### T1-04. 법률가 — 계약서 변경 추적

- **한 줄 설명(방향)**: "계약서 47번째 조항, 누가 언제 고쳤는지 추적. blame 한 번이면 변경 이력이 줄 단위로. 실수로 삭제한 조항도 checkout으로 복구."
- **보여주는 기능**: blame(줄별 변경 추적), revert(조항 되돌리기), branch(대안안 작성).
- **연결 튜토리얼**: T05(blame), T11(undo·revert).
- **일러스트**: 계약서 문서에 형광펜과 화살표가 줄마다 찍힌 모습.

### T1-05. 행정직원 — 문서 개정 이력, 회의록

- **한 줄 설명(방향)**: "팀 회의록을 매주 고치는데, 3개월 전 결정사항 찾기? log로 한 번에. '누가 그때 그 안 냈어?'도 blame으로."
- **보여주는 기능**: log, blame, commit(매주 회의록 스냅샷).
- **연결 튜토리얼**: T03, T05.
- **일러스트**: 회의록 문서 더미 + 달력.

### T1-06. 음악가·작곡가 — 악보·가사 버전 관리

- **한 줄 설명(방향)**: "이 곡, 두 가지 편곡버전이 필요해? 브랜치로 평행 우주. 가사 한 줄 고쳤는데 예전 게 낫던 것 같아? checkout으로 3번째 버전으로."
- **보여주는 기능**: branch(대안 편곡), commit(가사/악보 스냅샷), tag(데모 v1, 정식 v1).
- **연결 튜토리얼**: T06, T10.
- **일러스트**: 악보와 음표들이 브랜치로 갈라지는 모습.

### T1-07. 요리사·레시피 개발자 — 레시피 반복 개선

- **한 줄 설명(방향)**: "소스 레시피 30번 실험. '7번째가 제일 맛있었는데?' → checkout으로. 비건 버전? 브랜치로 따로. 레시피 노트가 곧 사진첩."
- **보여주는 기능**: commit(레시피 스냅샷), branch(비건/일반 버전), tag(메뉴판 확정 v1).
- **연결 튜토리얼**: T03, T06, T10.
- **일러스트**: 냄비와 레시피 카드가 사진첩 페이지로.

### T1-08. 게이머 — 세이지·모드 버전 관리, 롤백

- **한 줄 설명(방향)**: "게임 세이지 파일을 commit처럼. '이 파트 깼을 때로 돌아가자'. 모드도 버전별로. 마인크래프트 월드 히스토리도 이렇게."
- **보여주는 기능**: commit(세이지 스냅샷), checkout(롤백), branch(대안 플레이).
- **연결 튜토리얼**: T03, T11(reset).
- **일러스트**: 게임 컨트롤러 + 세이지 파일 아이콘들이 시간줄에.

### T1-09. 닷파일 (Dotfiles) — 여러 컴퓨터 설정 동기화

- **한 줄 설명(방향)**: "집 PC, 회사 PC, 노트북 — 세 컴퓨터의 설정(.vimrc, .zshrc 등)을 GitHub 저장소 하나로 동기화. 한 곳 고치면 push, 다른 곳에선 pull."
- **보여주는 기능**: remote, push, pull, clone, .gitignore(비밀 파일 제외).
- **연결 튜토리얼**: T02(clone), T09(remote), T13(.gitignore).
- **일러스트**: 세 대의 컴퓨터가 구름(GitHub)을 통해 연결.

### T1-10. 블로거·뉴스레터 작가 — 글 버전 관리 + GitHub Pages 발행

- **한 줄 설명(방향)**: "글 초고를 커밋으로. 발행은 GitHub Pages로 — GitHub Actions가 push만 하면 자동으로 웹사이트 업데이트. 호스팅비 0원."
- **보여주는 기능**: commit, GitHub Pages, Actions(자동 발행).
- **연결 튜토리얼**: T03, T18(Actions·Pages).
- **일러스트**: 펜과 종이 → 구름 → 웹사이트 화살표.

### T1-11. 이력서·포트폴리오 — GitHub Pages 이력서 + profile README

- **한 줄 설명(방향)**: "이력서를 GitHub Pages로 무료 호스팅. 내 GitHub 프로필(profile README)을 자기소개서로 꾸미면, 채용 담당자가 내 활동을 한눈에. contribution 그래프는 곧 성실도 증명."
- **보여주는 기능**: GitHub Pages, profile README, contribution graph, repo(public 작업물).
- **연결 튜토리얼**: T18(Pages), T21(profile README).
- **일러스트**: 이력서가 웹사이트로 변신 + 프로필 페이지.

### T1-12. 일기·가계부 — 개인 데이터 버전 관리

- **한 줄 설명(방향)**: "일기/가계부를 마크다운으로, GitHub에 비공개 저장소로. 10년 치 기록이 줄별로 추적 가능. '작년 이맘때 뭐 했지?' log로."
- **보여주는 기능**: private repo, commit, log, branch(연도별).
- **연결 튜토리얼**: T02(repo 생성), T03, T05.
- **일러스트**: 달력 + 일기장이 사진첩으로.

### T1-13. 교사 — 강의자료·시험지 버전 관리, 학생 과제 PR

- **한 줄 설명(방향)**: "강의자료 매년 업데이트, 학생들이 과제를 PR로 제출하면 코멘트로 피드백. 학생도 '버전 관리'를 자연스럽게 배우고, 교사는 채점 이력이 남아요."
- **보여주는 기능**: PR(과제 제출), review(피드백), commit(자료 개정), branch(연도/학기별).
- **연결 튜토리얼**: T16(issues-pr), T17(review).
- **일러스트**: 교탁 + 학생 과제 카드가 PR 형태로.

### T1-14. 번역가 — 번역 메모리 버전 관리

- **한 줄 설명(방향)**: "원문과 번역문을 줄별로 대응시켜 저장. 같은 문장이 다른 글에 나오면 checkout으로 재사용. '이 문장 3년 전에 어떻게 번역했지?' blame으로."
- **보여주는 기능**: blame, checkout, branch(번역 방향별), commit.
- **연결 튜토리얼**: T05, T06, T11.
- **일러스트**: 두 언어 문서가 화살표로 대응.

## 3. Tier 2 — 전문가/오픈소스 사례

> "Git/GitHub가 실제 산업·학계·공공을 어떻게 움직이나."

### T2-01. 리눅스 커널 — Git의 탄생지

- **한 줄 설명(방향)**: "Git 자체가 리눅스 커널을 관리하려고 Linus Torvalds가 2005년에 만든 거예요. 전 세계 수천 명의 개발자가 매일 수만 개의 커밋을 보내는, 가장 거대한 Git 사용처."
- **보여주는 기능**: 대규모 분산 워크플로우, merge, email 기반 패치(커널 특유), tag(커널 버전).
- **연결 튜토리얼**: T01(Git 탄생), T09(remote), T20(workflow models).
- **일러스트**: 리눅스 펭귄 + 거대한 commit 그래프.

### T2-02. Trunk-based Development — Google·Meta 규모

- **한 줄 설명(방향)**: "구글·메타 같은 초대규모 기업은 모두가 main(trunk)에 직접 작업. 브랜치를 아주 짧게(몇 시간) 쓰고 빠르게 통합. '클라우드 규모'의 워크플로."
- **보여주는 기능**: trunk-based workflow, 잦은 통합, 자동화된 CI.
- **연결 튜토리얼**: T20(workflow models).
- **일러스트**: 수천 명이 한 줄기(main)로 몰리는 모습.

### T2-03. GitFlow 원조 — Vincent Driessen 2010

- **한 줄 설명(방향)**: "2010년 Vincent Driessen이 블로그에 올린 '성공적인 Git 브랜치 모델'이 GitFlow의 원조. main/develop/feature/release/hotfix의 다계층 구조. 10년 넘게 업계 표준이었어요."
- **보여주는 기능**: GitFlow 워크플로우, multi-branch model.
- **연결 튜토리얼**: T20.
- **일러스트**: GitFlow 다이어그램(5개 브랜치 층).

### T2-04. 모노레포 (Monorepo) — Google·Meta·Microsoft

- **한 줄 설명(방향)**: "회사 전체 코드를 하나의 거대한 저장소에서. 구글은 수십억 줄 코드가 한 repo라서. 의존성 관리·일관된 도구ing이 강점, 대신 특수 도구(Bazel 등) 필요."
- **보여주는 기능**: 단일 저장소 대규모 관리, sparse-checkout, CODEOWNERS.
- **연결 튜토리얼**: T15(repo), T17(CODEOWNERS).
- **일러스트**: 하나의 거대 폴더 안에 수천 프로젝트.

### T2-05. 오픈소스 기여 흐름 — fork→PR→review→merge

- **한 줄 설명(방향)**: "전 세계 누구나 오픈소스에 기여할 수 있는 이유. fork로 내 계정으로 복사, branch·commit, PR로 제안, 메인테이너가 review 후 merge. 리눅스·React·TensorFlow 모두 이렇게."
- **보여주는 기능**: fork, PR, review, branch protection.
- **연결 튜토리얼**: T15(fork), T16(PR), T17(review).
- **일러스트**: fork → branch → PR → merge 흐름.

### T2-06. 정부·공공 데이터 — 18F, 미국 정부, 공공데이터

- **한 줄 설명(방향)**: "미국 18F(정부 디지털 서비스 팀)는 GitHub에서 공개 개발. 정부 코드·공공 데이터도 시민이 투명하게 볼 수 있어요. 한국 공공데이터도 일부 GitHub에."
- **보여주는 기능**: public repo, transparent development, open data.
- **연결 튜토리얼**: T14(social coding), T19(security in public).
- **일러스트**: 정부 건물 + 공개 코드 아이콘.

### T2-07. 학계 — 데이터 공유, arXiv + git, 오픈 저널

- **한 줄 설명(방향)**: "연구 데이터·코드·논문 초고를 GitHub 공개 저장소로. arXiv는 논문과 GitHub 코드를 연결. 재현 가능 연구의 표준이 되어가고 있어요."
- **보여주는 기능**: public repo, LFS(데이터), DOI 링크(Zenodo 연동).
- **연결 튜토리얼**: T13(LFS), T14.
- **일러스트**: 논문 + 데이터 + 코드가 한 저장소.

### T2-08. 책·문서 — GitBook, Pro Git (git-scm.com 자체가 git 관리)

- **한 줄 설명(방향)**: "이 사이트가 참고하는 Pro Git 책(git-scm.com/book) 자체가 GitHub에서 git으로 관리돼요. GitBook 같은 도구는 책을 git 저장소에서 자동 출판."
- **보여주는 기능**: docs-as-code, GitHub Pages, markdown.
- **연결 튜토리얼**: T13, T18(Pages).
- **일러스트**: 책이 GitHub 저장소에서 웹사이트로 변신.

### T2-09. "Oh Shit, Git!?!" — 실수 복구 현상

- **한 줄 설명(방향)**: "ohshitgit.com이라는 사이트가 유명해요. '아 망할, Git에서 실수했어' 상황별 복구법을 모아둔. 전 세계 개발자가 공감하는 현상 — Git의 실수 공포를 유머로 풀어요."
- **보여주는 기능**: reset, reflog, revert, cherry-pick 복구 패턴.
- **연결 튜토리얼**: T11(undo — 본 사이트의 OhShitGit 시뮬레이터와 직접 연결).
- **일러스트**: 당황한 사람 + 구명조끼.

### T2-10. GitHub Actions CI/CD — 실제 팀 자동화

- **한 줄 설명(방향)**: "수백 명의 팀이 매일 수천 번 push해도, Actions가 자동으로 테스트·빌드·배포. '사람이 실수하는 부분'을 로봇이 대신. 현대 소프트웨어 개발의 표준."
- **보여주는 기능**: Actions, matrix, secrets, environments, CD.
- **연결 튜토리언**: T18.
- **일러스트**: 컨베이어 벨트에 상자(코드)가 흐르고 로봇이 검사.

### T2-11. 프로필 README + contribution 그래프 — 구직 활용

- **한 줄 설명(방향)**: "GitHub 프로필을 자기소개서로 꾸미고, contribution 그래프(잔디밭)는 성실도의 시각적 증거. 채용 담당자가 GitHub를 먼저 본다는 말이 있을 정도."
- **보여주는 기능**: profile README, contribution graph, public repo.
- **연결 튜토리얼**: T21.
- **일러스트**: 잔디밭 그래프 + 면접관.

### T2-12. Codespaces·Copilot — 현대 워크플로우 변화

- **한 줄 설명(방향)**: "Codespaces는 브라우저만 있으면 어디서든 개발 환경. Copilot은 AI가 코드를 짜주는 페어 프로그래머. 2020년대 Git/GitHub 워크플로우를 근본적으로 바꾸는 중."
- **보여주는 기능**: Codespaces, Copilot, cloud dev environment.
- **연결 튜토리얼**: T21.
- **일러스트**: 구름 안의 코드 에디터 + AI 로봇 옆에 사람.
