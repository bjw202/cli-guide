# 디자인 가이드 — 비개발자를 위한 Git/GitHub 멀티페이지 가이드

> 본 파일은 **디자인/자산 전략 기획** 이다. 시리즈 결속(SPEC-TMUX-GUIDE-001)을 위해
> 디자인 패밀리를 공유하되, 멀티페이지 구조·Git 특화 자산으로 확장한다.
> 실제 CSS 코드/에셋 파일은 구현 Phase에서 제작된다. 모든 에셋은 "비개발자 학습 가치" 정당성을 포함한다.

---

## 1. 톤앤매너 (Tone & Manner) — 시리즈 공유

> tmux 가이드(SPEC-TMUX-GUIDE-001)와 **완전히 동일한 톤**. 시리즈 결속의 핵심.

| 키워드 | 설명 |
|---|---|
| **Bright** | 밝고 환한. 어두운 검은 터미널 이미지 피하고, 흰/밝은 배경 위에 따뜻한 색 얹기. |
| **Friendly** | 친근한. "어려운 도구" 인식 지우고, "재미있는 장난감"처럼 다가감. |
| **Tutorial** | 한 단계 한 단계. 스크롤이 곧 학습 진행 바. |
| **Dribbble/Notion-grade** | 시각적 완성도. 정돈된 여백, 섬세한 일러스트, 부드러운 모션. |
| **Warm, not cold** | 차가운 기술 느낌 아니라 따뜻한 종이/나무/포스트잔 질감의 온기. |
| **Series cohesion** | tmux 가이드와 한눈에 "같은 시리즈"로 인식될 것. |

**안티 테마(피할 것)**:
- 다크 모드 기본 / 네온 / 사이버펑크 / "해커" 미학.
- GitHub 기본 다크 테마 베끼기 (따뜻한 튜토리얼 톤과 충돌).
- 정보 밀집. 여백 부족하면 비개발자는 바로 이탈.

## 2. 컬러 시스템 — 시리즈 공유 + Git 액센트

### 2.1 시리즈 공유 토큰 (tmux 가이드와 동일)

> "햇살 드는 방" — 따뜻하고 편안한.

| 역할 | 색상 | 한국어 이름 | 비고 |
|---|---|---|---|
| Background-base | `#FFFBF5` 계열 | "따뜻한 크림색" | 순백(#FFF)은 차가워 보여 미색으로. |
| Background-surface | `#FFFFFF` | "카드 하양" | 카드/박스 위에만 순백. |
| Primary | `#FF8A5B` 계열 | "따뜻한 코랄" | 시리즈 시그니처. |
| Accent (코드/명령어) | `#3D5AFE` 연한 톤 | "편안한 인디고" | 단축키/코드/명령어. monospace와 짝꿍. |
| Secondary | `#26A69A` 계열 | "민트 팁" | 팁 박스/성공 상태. |
| Warning | `#EF5350` 부드러운 톤 | "코랄 레드(주의)" | 주의 박스. 자극적 빨강 금지. |
| Text-primary | `#2D2A26` | "따뜻한 먹색" | 검정 대신 먹색. |
| Text-secondary | `#7A726B` | "회갈색" | 본문 보조 텍스트. |

### 2.2 Git/GitHub 사이트만의 액센트 (tmux와 구별)

> tmux 가이드가 **민트(`#26A69A`)** 를 시그니처로 썼다면, 본 사이트는
> **따뜻한 초록(`#2E7D52` 계열)** 을 Git/GitHub 시그니처로 쓴다.

| 역할 | 색상 | 한국어 이름 | 정당성 |
|---|---|---|---|
| Git 액센트 (사례 콜아웃 박스, "성공 병합", commit 노드) | `#2E7D52` 계열 | "따뜻한 초록" | GitHub의 브랜드 초록(`#1F883D`)을 오마주하되, 시리즈의 따뜻한 톤과 어우러지게 더 부드럽게. 시리즈 공유 팔레트 안에서 녹아들며, 동시에 tmux(민트)와 한눈에 구별. |
| GitHub 마을 액센트 (마을/협업 메타포 일러스트) | `#5B6CFF` 부드러운 톤 | "연한 인디고(마을)" | GitHub의 indigo 계열 뉘앙스. "사람들이 모이는 공간" 느낌. |

**왜 따뜻한 초록인가 (Justification)**:
- GitHub 공식 브랜드 초록(`#1F883D`)은 너무 또렷해서, 따뜻한 크림 배경과 부딪힘.
- 따뜻한 톤으로 낮춘 `#2E7D52`는 시리즈 팔레트(크림/코랄/민트)와 조화.
- 동시에 "성장", "안전한 병합", "성공" 같은 긍정 신호로 자연스럽게 읽힘.
- tmux의 민트(`#26A69A`)와 충분히 다른 휘도·색상으로 "같은 시리즈 다른 권" 전달.

### 2.3 side-note 박스 색 매핑 (시리즈 공유 5종 + 신규 1종)

| 박스 | 색상 |
|---|---|
| 비유 박스 | 따뜻한 노랑 `#FFF7E6` 배경 (시리즈 공유) |
| 역사 박스 | 빈티지 베이지 `#F5EFE0` (시리즈 공유) |
| 팁 박스 | 민트 `#E6F7F4` (시리즈 공유) |
| 주의 박스 | 부드러운 코랄 `#FDECEA` (시리즈 공유) |
| "알아두면 좋은" 박스 | 연한 라벤더 `#F1ECFB` (시리즈 공유) |

## 3. 타이포그래피 — 시리즈 공유

> tmux 가이드와 **동일 폰트 패밀리**. 결속 위해 변경 없음.

| 역할 | 폰트 | 이유 |
|---|---|---|
| 한국어 본문/헤드라인 | Pretendard / Spoqa Han Sans Neo 계열 | 가독성 좋은 현대적 한국어 sans. |
| 영문 본문 | Inter / Nunito Sans | 둥글고 부드러운 — 친근함. |
| **코드/명령어/SHA** | **JetBrains Mono / Fira Code** | monospace. `git status` / `a1b2c3d` 따위 표시. |
| 디스플레이 (큰 헤드라인) | 둥근 geometric sans (Pretendard Black / Nunito ExtraBold) | 튜토리얼의 부드러움. |

**위계(방향)**:
- Hero 헤드라인: 56~72px / 800 weight
- 페이지 헤드라인: 40~48px / 700
- 섹션 헤드라인: 28~34px / 700
- 서브헤드: 22~26px / 600
- 본문: 17~18px / 400 / line-height 1.7
- 명령어/코드: 15~16px monospace
- side-note: 15px / 400

**명령어 표시 컴포넌트("command pill")**:
- `git status` 따위를 둥근 알약 모양(`#3D5AFE` 톤 배경, monospace).
- tmux의 "key cap"과 짝을 이루지만, 명령어는 한 줄이라 알약 형태.

## 5. 핫 에셋 인벤토리 ("Hot Assets") — 시리즈 재사용 + Git 신규

> 각 에셋은 **비개발자 학습 가치**로 정당화되어야 한다. 과잉 엔지니어링 금지.

### A. Scroll-triggered Reveal Animations — 시리즈 재사용 (tmux에서 가져옴)
- **목적**: 스크롤하면 개념이 "나타나는" 모션. 페이지마다 적용.
- **비개발자 학습 가치**: 정보를 "한 입 크기"로 나눠줘 인지 부하 낮춤.
- **기술(방향)**: 순수 JS `IntersectionObserver` + CSS. tmux 사이트의 `reveal.js` 그대로 재사용.

### B. Interactive Fake-Terminal — 시리즈 재사용 + Git 확장
- **목적**: 가짜 터미널 박스. 버튼 클릭 → `git status` / `git add` / `git commit` 시퀀스 재생.
- **비개발자 학습 가치**: 실제 터미널 없이 "git이 어떻게 반응하는지" 체험.
- **기술(방향)**: tmux의 fake-terminal 패턴 재사용. git 명령어 스크립트 추가.
- **데모 시나리오**:
  1. `git init` → 빈 저장소 메시지.
  2. `git status` → 빨간/초록 파일 상태.
  3. `git add .` → 상태 변화 (빨강→초록).
  4. `git commit -m "..."` → 커밋 생성 메시지.
  5. `git log --oneline` → 커밋 리스트 출력.

### C. Git Graph Visualizer — 신규 (Git 핵심 자산)
- **목적**: 인터랙티브 commit 그래프. 브랜치가 가지 뻗고, 병합이 합류하는 모습 애니메이션.
- **비개발자 학습 가치**: **Git의 가장 큰 진입장벽인 "브랜치가 뭐고 병합이 뭔지"를 한 눈에**.
  선형 텍스트로 100번 설명하는 것보다 그래프가 한 번 움직이는 게 압도적.
- **기술(방향)**: 인라인 SVG 또는 positioned divs. 스크롤/클릭에 따라 commit 노드가 추가되고
  브랜치가 갈라지고 병합 지점이 생기는 애니메이션.
- **데모 시나리오**:
  1. main 브랜치 일직선.
  2. "feature 브랜치 만들기" → 가지 위로 갈라짐.
  3. "커밋 추가" → 각 가지에 노드.
  4. "merge" → 두 가지가 합류 노드 생성.
  5. (선택) "rebase" → feature 커밋들이 main 끝으로 재배치되는 모습.
- **주요 사용**: T06(branch), T07(merge), T08(rebase).

### D. Staging-area Flow Diagram — 신규
- **목적**: 3개의 방(working/staging/repo) 다이어그램. 파일이 방 사이를 움직이는 드래그/애니메이션.
- **비개발자 학습 가치**: "왜 add를 해야 commit하지?" "diff는 왜 3종류나 있지?"를 시각적으로 해소.
- **기술(방향)**: 3개 영역 CSS grid + 파일 아이콘이 add/commit에 따라 이동하는 모션.
- **주요 사용**: T04(staging).

### E. PR Workflow Animated Diagram — 신규 (GitHub 핵심 자산)
- **목적**: fork → branch → commit → push → PR → review → merge 흐름을 애니메이션으로.
- **비개발자 학습 가치**: PR이 뭔지 한 장의 그림+모션으로. "왜 fork하지?", "review는 언제?"를 흐름으로 파악.
- **기술(방향)****: 단계별 노드가 순차적으로 하이라이트되는 흐름도. 각 단계 클릭 시 상세 팝업.
- **주요 사용**: T16(issues-pr), T15(repo).

### F. Actions Pipeline Animation — 신규
- **목적**: event → job → step → artifact → deploy 파이프라인 애니메이션.
- **비개발자 학습 가치**: "CI/CD가 뭔지"를 로봇 공장 컨베이어 비유로 시각화.
- **기술(방향)**: 각 단계가 순차적으로 켜지는 흐름도. event(push) → runner → steps →
  artifact(빌드 결과) → deploy(배포) 흐름.
- **주요 사용**: T18(actions).

### G. "Oh Shit Git" Interactive Recovery Simulator — 신규 (시그니처)
- **목적**: 실수 상황을 고르면 → 복구 명령 + 설명이 나오는 인터랙티브.
- **비개발자 학습 가치**: 비개발자가 Git을 두려워하는 가장 큰 이유 = "실수하면 날아갈까".
  이 공포를 직접 시나리오로 체험하고 해소. ohshitgit.com 오마주.
- **기술(방향)**: 카드 선택 UI → 명령어 + "왜 이게 작동하나" 설명 카드.
- **시나리오 메뉴**:
  1. "잘못 브랜치에서 커밋했어요" → cherry-pick 또는 새 브랜치.
  2. "reset --hard 했는데 필요했어요" → reflog 복구.
  3. "커밋 메시지 오타 났어요" → `git commit --amend`.
  4. "방금 커밋 취소하고 싶어요" → `git reset HEAD~1`.
  5. "이미 push한 커밋 취소하고 싶어요" → `git revert`.
  6. "force push 해서 동료 작업 날아갔어요" → (경고 + 협조 안내).
  7. ".env 파일 실수로 push했어요" → filter-repo + 비밀번호 변경.
- **주요 사용**: T11(undo).

### H. Case-gallery Card Grid + Filter — 신규 (시그니처)
- **목적**: 사례 카드 그리드 + 필터(Tier/분야/난이도).
- **비개발자 학습 가치**: "아~ 이렇게도 쓸 수 있구나" — 사용자가 명시적으로 강조한 시그니처.
- **기술(방향)**: CSS grid 카드 + JS 필터. 각 카드: 따뜻한 일러스트 + 한 줄 설명 + 관련 git/github 기능 + "관련 튜토리얼" 링크.
- **주요 사용**: cases/index.html.

- **기술(방향)**: 두 버전 side-by-side + 라디오/토글 선택 + 결과 미리보기.

### O. Warm Friendly Illustration Set — 시리즈 재사용 + Git 테마 추가
- **목적**: 핵심 비유(사진첩/평행 우주/마을/컨베이어)를 따뜻한 일러스트로.
- **비개발자 학습 가치**: 추상 개념이 일러스트로 "구체적 경험". 기억에 오래 남음.
- **필수 일러스트(Git/GitHub 테마)**:
  - 시간을 되돌리는 사진첩 (Concept T01)
  - 3개의 방: 작업/대기/기록 (Staging T04)
  - 평행 우주가 갈라지는 나무 (Branch T06)
  - 두 세계가 합쳐지는 합류점 (Merge T07)
  - 구름 위의 마을 = GitHub (Concept T14)
  - PR 흐름: 사람들이 게시판에 글 올리고 도장 찍는 모습 (PR T16)
  - 로봇 비서가 컨베이어에서 상자 옮기는 모습 (Actions T18)
  - "Oh Shit Git" 안심 일러스트: 당황한 사람 + 구명조끼 (Undo T11)
- **스타일**: tmux 일러스트와 동일 톤. 플랫 + 약간의 텍스처, 따뜻한 팔레트, 둥근 형태.

## 6. 접근성 & 모바일 (Accessibility & Mobile) — 시리즈 공유 + 멀티페이지 확장

| 항목 | 방향 |
|---|---|
| 키보드 탐색 | 모든 인터랙티브 요소 tab 가능. 데모 버튼 Enter/Space. 검색 Ctrl+K. |

| 모바일 내비 | 1단 레이아웃. 헤더 햄버거 메뉴. 사이드바 TOC 상단 접기. 튜토리얼 하단 바(prev/next). |

| 한국어 줄바꿈 | `word-break: keep-all`로 단어 중간 끊김 방지. |
| 페이지 간 일관성 | 모든 페이지에 동일 글로벌 헤더/푸터. 모션·톤 일관. |
| 검색 접근성 | 검색 결과 키보드 탐색 가능. 스크린 리더 결과 개수 안내. |
| 일러스트 대안 | 핵심 일러스트는 `aria-describedby` 또는 인접 텍스트로 의미 전달 보장. |
